# RD-02: เป้าหมายผลิตภัณฑ์ (Product Goals)

**สถานะ:** Draft  
*(เดิมชื่อ business-requirements — ใช้เป็นเป้าหมายผลิตภัณฑ์เกมอิสระ/พอร์ตโฟลิโอ ไม่ใช่แผน monetize เว็บ)*

---

## 1. วัตถุประสงค์ผลิตภัณฑ์

ส่งมอบเกม Terminal ที่:

1. **เล่นสนุกได้จริง** ในช่วงสั้นและยาว (สำรวจ–สู้–เติบโต–เซฟ)  
2. **ขยายคอนเทนต์ถูก** ด้วยไฟล์ data  
3. **แสดงฝีมือออกแบบระบบ** (ความรู้ซ่อน, ความเสี่ยง, ป้องกันแพ้ทาง, การ์ด)  
4. **พัฒนามีวินัย** ภายใต้ workspace แยก core / design / projects  

## 2. ไม่ใช่เป้าหมาย v1

- รายได้จากโฆษณา/IAP  
- DAU/SEO  
- กราฟิก AAA  

## 3. เกณฑ์สำเร็จ (ผลิตภัณฑ์)

ดู RD-01 §6 และ Milestone M1–M6 ใน RD-24

## 4. ผู้มีส่วนได้ส่วนเสีย

| บทบาท | ความสนใจ |
|--------|----------|
| Product owner (ผู้โปรเจกต์) | วิสัยทัศน์ เกมสนุก แผนเฟส |
| ผู้เล่น | เล่นรู้เรื่อง เซฟได้ ท้าทายแบบยุติธรรม |
| AI/Dev ภายใต้ Z-MOS | ขอบเขตชัด FR ชัด ทำทีละเฟส |
