# RD-07: โครงสร้างจอ / โหมด Terminal

**สถานะ:** Draft  
**ชื่อเดิม:** โครงสร้างหน้าเว็บไซต์ → **เปลี่ยนเป็นจอเกมข้อความ**

---

## 1. วัตถุประสงค์

กำหนด “จอ” (screen/mode) ของเกมแทนโครงสร้างเว็บ

## 2. แผนที่จอ

```text
[Boot]
   └─ [Main Menu]
         ├─ เลือกโลก (P10)
         ├─ เลือกตัว / สร้างตัว
         └─ แอดมิน (ซ่อน/flag)
              │
              ▼
         [Field / สนาม]
              ├─ สำรวจ / เดินทาง
              ├─ ดูสิ่งรอบตัว / เข้าหา
              ├─ คลัง · ร้าน · journal · ดูภาพ
              ├─ ออโต้ (P9)
              └─ เซฟ / ออก
              │
              ├─► [Combat]
              │      ├─ แอคชันผู้เล่น
              │      └─ เฟสป้องกัน
              ├─► [Chest resolve]
              ├─► [NPC / Master dialog]
              └─► [Shop]
```

## 3. รายการจอ

| จอ | อินพุตหลัก | เอาต์พุตหลัก |
|----|------------|--------------|
| Main Menu | หมายเลข | เข้าโลก/ตัว |
| Character Create | ข้อความ + เลข | ตัวใหม่ |
| Field | เลขคำสั่ง | สถานะ + รายการรอบตัว |
| Combat | เลขสกิล/คอมโบ/กัน | โลดาเมจ ผลแพ้ชนะ |
| Inventory/Equip | เลข | ใส่ของ/การ์ด |
| Shop | เลข | ซื้อขาย |
| Journal | เลข | ความรู้ที่ปลดแล้ว |
| Art Viewer | เลข | ASCII |
| Pause (Auto) | เลขตัดสินใจ | กลับ Field/Combat |
| Admin | คำสั่ง | รีเซ็ต/สร้างโลก |

## 4. กฎนำทาง

- ออกจากจอย่อยกลับ Field หรือเมนูก่อนหน้าเสมอ มี `0` = กลับ  
- ตาย/จบรอบ → ข้อความชัด → เมนูหรือโหลดเซฟ  
- ไม่มี URL / routing เว็บ
