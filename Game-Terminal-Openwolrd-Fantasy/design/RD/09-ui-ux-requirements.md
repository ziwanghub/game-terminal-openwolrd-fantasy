# RD-09: UI/UX Requirements (Terminal)

**สถานะ:** Draft  

---

## 1. วัตถุประสงค์

กำหนดประสบการณ์ข้อความให้อ่านง่าย สนุก และพิเศษตอนเหตุการณ์สำคัญ โดย **ไม่มีกราฟิกจริง**

## 2. หลักการ UX

| หลัก | คำอธิบาย |
|------|----------|
| โครงคงที่ | สถานะ/เมนู/ไฟต์ วางตำแหน่งสม่ำเสมอ |
| ปกติเรียบ เหตุการณ์เด่น | ไม่ spam ASCII ทุกเทิร์น |
| ผู้เล่นเลือกดูภาพ | เมนูดูมอน/การ์ด/ไอเทม |
| ข้อมูลสำคัญก่อนสวย | HP/MP/ตัวเลือกอ่านก่อนศิลป์ |
| ปิดของรกได้ | prefs ปิดรูปใน combats, ปิด typewriter |

## 3. ระดับการแสดงผล

| ระดับ | ชื่อ | ใช้เมื่อ |
|-------|------|----------|
| L0 | Compact | ออโต้, ระหว่าง tick โลก |
| L1 | Standard | เมนูสนาม, คลัง, ร้าน |
| L2 | Spotlight | legendary, บอส first kill, การ์ดหายาก |

## 4. ข้อกำหนด UI

| ID | ข้อกำหนด |
|----|----------|
| UX-01 | แถบ HP/MP แบบตัวอักษร (เช่น ████░░) |
| UX-02 | กล่องสถานะ L1 แสดงครบ vitals เงิน ที่ สกิล พร/สถานะ |
| UX-03 | ไฟต์มีบล็อก “ล่าสุด” สรุปดาเมจเทิร์นก่อน |
| UX-04 | เมนูเลือกเป็นหมายเลขชัดเจน |
| UX-05 | L2 แสดง ASCII จาก `data/art/` กลางจอ + สเปกของ |
| UX-06 | ??? ใช้ silhouette ไม่โชว์ art จริง |
| UX-07 | ความกว้างเป้าหมาย 72–80 คอลัมน์; มี art เวอร์ชัน small |
| UX-08 | ภาษาหลักของ UI: ไทย (ข้อความระบบ) |
| UX-09 | ไม่บังคับสี; สี (Rich) เป็น optional ช่วง polish |
| UX-10 | Typewriter delay เฉพาะ L2 และปิดได้ (default แนะนำปิดหรือ 0) |

## 5. จอหลักที่ต้องมี

1. เมนูเริ่ม (โลก/ตัวละคร/สร้างใหม่)  
2. สนาม (สถานะ + สิ่งรอบตัว + คำสั่ง)  
3. ต่อสู้ (+ เฟสป้องกัน)  
4. คลัง / อุปกรณ์ / การ์ด  
5. ร้าน  
6. Journal ความรู้  
7. ดูภาพ (art viewer)  
8. ยืนยันเซฟ/ออก  

## 6. Anti-patterns (ห้าม)

- ASCII เต็มจอทุกมอนทุกเทิร์น  
- ซ่อนตัวเลือกสำคัญในศิลป์  
- บังคับ delay ยาวตอนออโต้  
- แสดงเลเวลมอน / ตารางธาตุเต็มก่อน unlock
