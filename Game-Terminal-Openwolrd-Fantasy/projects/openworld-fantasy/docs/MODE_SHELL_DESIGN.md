## 1. ปัญหาที่เจอตอนเล่น

ตอนนี้สนามโชว์ตัวเลือกหนา:

```text
1–9  +  S P C L N K Y B U H T  +  ใบ้แต้ม
```

รวมกับกระเป๋าที่มี **ร้าน / ตลาด / กระดาน** ซ้อนใน **5**  
→ มือใหม่ต้องจำหลายคีย์ · โหมดไฟต์กับสนามปนในหัว · “แทบหลัก” ไม่ชัด

**chrome โหมดมีอยู่แล้วบางส่วน** (`〔สนาม〕` / `〔ไฟต์〕` / ดันเจียน) แต่**เมนูยังไม่ผูกโหมด**

---

## 2. ข้อเสนอ (ผู้เล่น) — เห็นด้วยในหลัก

แบ่ง **โหมด (Mode)** แล้วเมนูเปลี่ยนตามโหมด  
มี **แทบหลัก (Base strip)** ที่เกี่ยวกับตัวละครเสมอ (หรือกดเข้าได้ทุกโหมดที่อนุญาต)

| โหมด | หน้าที่ | ตัวอย่างแอคชัน |
|------|---------|----------------|
| **PERSONAL** | จัดการตัวเอง · ดูข้อมูลผู้เล่น | สถานะ · กระเป๋า · สวมใส่ · ภารกิจที่รับ · เงิน · แต้ม P/N · เซฟ |
| **EXPLORE** (สนาม/สำรวจ) | ดำเนินเรื่อง · โลก | พัก · สำรวจ · เข้าหา · เดินทาง · สายตา · พูด · หีบ · ออโต้ |
| **COMBAT** | ต่อสู้ | โจมตี · คอมโบ · ยา · หนี · ปาร์ตี้ · สติ/ATB · แท่งจำเป็น |
| **SHOP** | ซื้อขาย | ร้าน NPC · (แยก) ตลาด P2P · คราฟ optional |

นี่คือ **Mode Shell** — ชั้น UX ไม่ใช่ rewrite logic domain

---

## 3. ความเห็นสถาปนิก (แนะนำ)

### เห็นด้วย — ควรทำ

1. **ลด cognitive load** — เล่นแล้วเมนูยาวเกินจริง  
2. **สอดคล้องโค้ดที่มี** — แยก `field_*` / `combat_session` / `shop` / `bag` อยู่แล้ว แค่ **จัดทางเข้า**  
3. **แทบ PERSONAL เป็น “บ้าน”** — เงิน · เควสที่รับ · ของ · สถานะ ควรรวมที่เดียว  
4. **SHOP แยกจาก EXPLORE** — ตอนนี้ร้านอยู่ใน 5 กับ NPC ปนกัน มึน  

### ข้อควรระวัง

| ความเสี่ยง | แนวทาง |
|------------|--------|
| สลับโหมดเยอะจนช้า | โหมดหลักมี 3–4 · ไม่แตก micro-mode |
| จำ “อยู่โหมดไหน” | chrome ชัด: `〔สำรวจ〕〔ตัวละคร〕〔ไฟต์〕〔ร้าน〕` + 1 บรรทัดคำสั่งโหมด |
| ไฟต์ต้องเปิด PERSONAL ยา | COMBAT เก็บ **ยา/ล้าง** ไว้ · PERSONAL ลึกตอนนอกไฟต์ |
| ตลาด vs ร้าน | SHOP มีแท็บย่อย: ร้าน NPC · ตลาดผู้เล่น · (คราฟ) |
| มือเก่าติดปุ่มเดิม | รอบแรก: **เลขโหมด + hotkey เดิมยังใช้ได้** (alias) |

### ความเห็นสรุป

> **ทิศทางถูกและคุ้มทำ** — ไม่ใช่ระบบใหม่ทั้งเกม แต่เป็น **UX shell**  
> ควรทำเป็นเฟส อย่า rewrite field_loop ทั้งก้อนในครั้งเดียว  
> ผลที่ต้องการ: ทุกจอมี **(A) แทบสถานะโหมด (B) เมนูสั้นของโหมด (C) hotkey ลึก optional**

## 4. โครงจอที่เสนอ (Terminal)

### 4.1 โครงร่วมทุกโหมด

```text
〔โหมด〕 รายละเอียดสั้น (พื้นที่ / ศัตรู / ชื่อร้าน)
┌─ L1c สถานะย่อ (โหมดกำหนดว่าโชว์อะไร) ─┐
│ ชื่อ Lv อาชีพ · HP/MP · (เงินถ้า PERSONAL/SHOP) │
│ ATB ถ้า COMBAT · ภารกิจย่อถ้ามี active          │
└────────────────────────────────────────┘
── เมนูโหมด (สั้น 4–8 ตัวเลือก) ──
 1 …  2 …  3 …
 [I] ตัวละคร  [?] ช่วย   (แทบหลัก / ข้ามโหมดที่อนุญาต)
เลือก:
```

### 4.2 แทบหลัก PERSONAL (ปุ่มแนะนำ: **I** หรือ **Tab จำลอง = I**)

เข้าได้จาก EXPLORE / SHOP / หลังไฟต์  
**ห้าม** เปิดเต็มระหว่าง COMBAT (ยกเว้นยาด่วนที่ฝังใน COMBAT อยู่แล้ว)

| # | เมนู PERSONAL | ผูกของเดิม |
|---|----------------|------------|
| 1 | สถานะเต็ม | S |
| 2 | กระเป๋า (ของใช้/วัตถุดิบ/การ์ด) | bag hub หมวด |
| 3 | สวมใส่ / เกียร์ | equip panel |
| 4 | ภารกิจที่รับ (โลก + กระดาน active) | 9 + board active |
| 5 | เงิน / เศรษฐกิจย่อ | money lines + inbox |
| 6 | แต้ม P / นิสัย N / อาชีพ C | P N C |
| 7 | สกิล K · ปาร์ตี้ Y · ห้องสมุด L | K Y L |
| 8 | ตั้งค่า U · เซฟ | U · 7 |
| 0 | กลับโหมดก่อนหน้า | |

| 2 | สำรวจ | 2 |
| 3 | เข้าหา / ใช้สายตา | 3 · f_/o_/talk_ |
| 4 | เดินทาง | 4 |
| 5 | เข้าสู่ PERSONAL | (ใหม่ทางเข้า) |

## 5. แผนที่โหมด (state)

```text
          ┌────────────┐
          │ WORLD MENU │
          └─────┬──────┘
                ▼
          ┌────────────┐
     ┌───►│  EXPLORE   │◄───┐
     │    └─┬───┬───┬──┘    │
     │      │   │   │       │
     │   combat shop I      │
     │      │   │   │       │
     │      ▼   ▼   ▼       │
     │  COMBAT SHOP PERSONAL│
     │      │   │   │       │
     └──────┴───┴───┘───────┘
              win/flee/0
```

**กฎสลับ**

- EXPLORE → COMBAT: ระบบเท่านั้น (ผู้เล่นไม่กด “เข้าโหมดไฟต์” เอง)  
- EXPLORE → SHOP: ผู้เล่นหรือ NPC  
- * → PERSONAL: ผู้เล่นกด **I** (หรือ 5 ใน EXPLORE = ทางลัด PERSONAL)  
- COMBAT → PERSONAL เต็ม: **ไม่อนุญาต** (ยาอยู่ใน 3)  
- ทุกโหมด 0: กลับชั้นบนตาม stack

## 6. แผน implement (เฟส — ไม่งานก้อนเดียว)

### Phase A — Shell เบา (ROI สูง · 1–2 รอบ)

1. นิยาม `GameMode = explore | personal | combat | shop`  
2. `render_mode_actions(mode, ctx)` แทน `render_field_actions` ยาว  
3. EXPLORE เมนูสั้น · **I** = PERSONAL submenu (ย้าย P/N/S/K/Y/L/U/bag เข้าไป)  
4. Hotkey เดิมยัง alias ได้ 1 เวอร์ชัน  

### Phase B — แยก SHOP

1. ร้าน/ตลาดไม่อยู่ใต้ bag เป็นหลัก — เข้าจาก EXPLORE 6 หรือ NPC  
2. Bag ใน PERSONAL = ของผู้เล่นอย่างเดียว  

### Phase C — COMBAT chrome ล็อก

1. ยืนยันเมนูไฟต์สั้น + ATB strip only  
2. หลังชนะ panel แล้ว pop กลับ EXPLORE ชัด  

### Phase D — เอกสาร + tutorial

1. H/T อัปตามโหมด  
2. Doc Policy: อัป ARCHITECTURE §4 แถว Mode Shell  

**ยังไม่ทำใน A–C:** rewrite domain, Z-MOS เต็ม, multi-enemy พร้อมกัน
