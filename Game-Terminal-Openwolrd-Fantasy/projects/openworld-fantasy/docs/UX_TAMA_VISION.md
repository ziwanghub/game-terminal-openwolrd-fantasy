## 0. เป้าหมายหนึ่งประโยค

ให้ตัวละคร (และอาจเพื่อนปาร์ตี้) **รู้สึกมีชีวิตแบบทามาก๊อต** บน terminal:  
จอสถานะกลาง + แถบ soft + เมนูสั้น  
ค่าสถานะ**ลด/เพิ่มตามสถานการณ์เกมและเวลา** — ไม่โชว์สูตรดิบ

**ไม่** เปลี่ยนทั้งเกมเป็นแค่เลี้ยงสัตว์ — RPG โลกเปิดยังเป็นหลัก

## 1. หลักการออกแบบ (ล็อกไว้)

| # | หลัก | ความหมาย |
|---|------|----------|
| T-P1 | **RPG หลัก + Tama ที่จอสถานะ** | สำรวจ/ไฟต์/ร้าน ยัง Mode Shell · Tama เน้น PERSONAL / แถบย่อ |

## 2. เวลาใน Terminal — ทำได้แค่ไหน

| แบบ | ความหมาย | เฟส |
|-----|----------|-----|
| **A. Game tick** | พัก/สำรวจ/เดินทาง/ไฟต์ → `game_time++` → อัป needs | **T0–T1 หลัก** |
| **B. Load delta** | เปิดเกมเทียบ `saved_at` กับนาฬิกาเครื่อง → จำลอง “ไม่ได้ดูแล” | **T1** |
| **C. Live refresh** | นั่งค้างจอ แถบขยับเอง (non-blocking input) | **T3 optional** |
| **D. 24h แม้ปิดเกม** | ได้ด้วย B ไม่ต้อง process ค้าง | ครอบคลุมด้วย T1 |

**สูตรที่ล็อก:**  
`needs = f(action เกม, เวลาที่ผ่านตอนโหลด)`  
ไม่ใช่แค่ `f(นาฬิกาอย่างเดียว)`

## 3. ชั้นสถานะแนวทามาก๊อต (Needs)

ชื่อ UI เป็น soft — คีย์ภายในซ่อน/ภาษาอังกฤษในโค้ด

| UI (ตัวอย่าง) | เพิ่มเมื่อ | ลดเมื่อ | หมายเหตุ |
|---------------|-----------|---------|----------|
| **ท้อง / เสบียง** | กิน เสบียง ร้าน | สำรวจ เดินทาง ไฟต์ยาว | ผูก consumable |
| **ล้า** | พัก เมือง | เดินทาง ดัน ไฟต์ต่อ | ใกล้ pressure กลับด้าน |
| **จิต / สติ** | พัก ชา เวลาเบา | ใช้สติไฟต์ / ทางเลือก ★ | มี intel อยู่แล้ว |
| **อารมณ์ / ขวัญ** | ชนะ เควส ปาร์ตี้ | แพ้ ซุ่ม | soft label |
| **ความกดดัน** | (ของเดิม) | พัก | อยู่แล้ว — รวมในแถบ Tama ได้ |

ทุกค่า clamp + soft band โชว์ ไม่โชว์ตัวเลขดิบใน UI ปกติ

## 5. แผนเฟส implement (T0–T3)

**ยังไม่เริ่มโค้ด** จนกว่าจะจัดลำดับหลัง playtest / ก่อนหรือคู่ขนาน W0 ตามทีม

| เฟส | ชื่อ | ขอบเขต | สถานะ |
|-----|------|--------|--------|
| **T0** | Needs data + tick | ฟิลด์ needs บน player · อัปจาก rest/explore/travel/combat/eat · soft labels | **1.13.15** |
| **T1** | Load delta + แถบ UI | `saved_at` · จำลองตอนโหลด · soft notes | **1.15.0** |
| **T2** | Tama panel เต็ม | ASCII · R พัก / E กิน ใน PERSONAL · mood | **1.15.0** |
| **T3** | Live optional | refresh เบาตอนเปิด panel / enter anim | **1.20.0** |

### T0 — รายละเอียด

- `player["needs"]` หรือฟิลด์ชัด: hunger, fatigue, morale (+ ใช้ intel/pressure ที่มี)
- ตาราง event → delta (YAML หรือ constants ใน domain)
- เทส unit: rest ลดล้า, explore เพิ่มล้า/หิว

### T1 — รายละเอียด

- เซฟเก็บ `saved_at` (ISO / unix)
- ตอน `load_player`: `delta_hours` → soft apply (cap สูงสุดต่อครั้ง)
- `render_needs_bars` ใน ui_terminal
- ไม่ฆ่าตัวละครจากห่างนานเกิน (soft only)

### T2 — รายละเอียด

- `personal_hub` โหมดแสดง Tama frame
- เมนู 2 กิน = ใช้ consumable กลุ่ม food
- mood flavor จาก needs band + personality soft

### T3 — รายละเอียด (optional)

- animation เบา 2–3 เฟรมตอนเข้า PERSONAL
- live tick เฉพาะเมื่ออยู่ Tama panel (ไม่บังคับทั้งสนาม)
