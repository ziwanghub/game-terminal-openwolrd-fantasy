## 0. เป้าหมายหนึ่งประโยค

แต่ละ**โลก**รันเหมือนเซิร์ฟกลาง: มีประวัติผู้เล่น·ไอเทม·มอน·เรื่องที่ผู้เล่นขับเคลื่อน  
ผู้เล่นคนอื่นเข้ามาอาจ**เจอเงาจากข้อมูลจริง**ของคนในโลก (สู้ / คุย / ทีม)  
มี**จัดอันดับโลก**โชว์แค่ชื่อ·แรงก์·ฉายา — ที่ 1 ≠ ดาเมจสูงสุดเสมอ — ท้าอันดับได้โดย**จ่ายค่าหัว**

---

## 1. หลักการออกแบบ (ล็อกไว้)

| # | หลัก | ความหมาย |
|---|------|----------|
| W-P1 | **World-as-server** | โลก = state ร่วม ไม่ใช่แค่โฟลเดอร์เซฟเดี่ยว |
| W-P2 | **Player-as-NPC (echo)** | เจอผู้เล่นอื่น = ใช้ความสามารถ**จริงจากเซฟ/โปรไฟล์** |
| W-P3 | **Snapshot ไม่ทำร้ายเซฟเจ้าของ** | สู้/คุยกับเงา**ไม่** soft-death หรือแก้ของในเซฟเขา |
| W-P4 | **Anti-spoiler อันดับ** | บอร์ดโชว์ชื่อ·แรงก์·ฉายา เท่านั้น — คะแนน/สูตรซ่อน |
| W-P5 | **ที่ 1 ≠ เก่งสุดแบบดิบ** | คะแนนซ่อนจากหลายแกน: เอาตัวรอด ภารกิจ ฉลาดเล่น ฯลฯ |
| W-P6 | **ท้าอันดับ = จ่ายค่าหัว** | ขอท้าจ่ายเงินให้เจ้าของ (หรือกอง+เจ้าของ) · แพ้เสียค่า · ชนะได้ที่นั่งโชว์ |
| W-P7 | **Async ก่อน realtime** | เงาจากเซฟล่าสุดก่อน · online จริงทีหลัง |
| W-P8 | **ไม่ spoiler เงื่อนไขขึ้นบอร์ด** | เหมือน special mission — ผู้เล่นประเมินเอง |

## 2. แยกตัวตนสามอย่าง (สำคัญ)

| ชนิด | คืออะไร | ใช้เมื่อ |
|------|---------|----------|
| **Live player** | คนที่กำลังเล่น session นี้ | I/O จริงของ client |
| **World ghost / echo** | snapshot ความสามารถจากเซฟในโลกนั้น | เจอในสนาม · สู้ · คุย · ชวนทีม |
| **Rank card** | ชื่อ · แรงก์ soft · ฉายา (ถ้ามี) | บอร์ดอันดับโลก |

ข้อความ UI ควรสื่อว่าเป็น**เงา/ร่องรอย** จนกว่าจะมี online จริง  
ตัวอย่าง soft: *「เงาของนักเดินทางที่เคยผ่านที่นี่」*

## 3. สรุประบบที่ต้องการ

### 3.1 โลกรันเป็นเซิร์ฟ

- เลือกโลกตอนเข้า (default / hardcore / nightmare / social ฯลฯ)
- แต่ละโลกมี: ผู้เล่นที่เคยเข้า, ตลาด, อันดับ, (optional) event log
- เรื่อง/เศรษฐกิจดำเนินจากพฤติกรรมผู้เล่นจริงในโลกนั้น

### 3.2 ข้อมูลผู้เล่นในโลก

- ผู้เล่น 1 เล่น → ข้อมูลจริงถูกเก็บใน world state
- ผู้เล่น 2 เข้าโลกเดียวกัน → อาจเจอ echo ของผู้เล่น 1
- ต่อสู้ / โต้ตอบ / ร่วมทีม → ใช้**ความสามารถจริงตาม snapshot** ของผู้เล่น 1  
  **ไม่** แก้ HP/ของในเซฟผู้เล่น 1 จากการที่เงาถูกสู้

### 3.3 จัดอันดับโลก

| โชว์ | ไม่โชว์ |
|------|---------|
| ลำดับ, ชื่อ, ฉายา, แรงก์ soft | คะแนนดิบ, HP, ATK, สกิล, เงิน, checklist เงื่อนไข |

- ขึ้นบอร์ดเมื่อผ่าน**เงื่อนไขซ่อน** (multi-factor soft)
- อันดับ 1 อาจมาจากเอาตัวรอด / ภารกิจ / เล่นฉลาด ไม่ใช่ดาเมจสูงสุด

### 3.4 ท้าสู้แย่งอันดับ

| เหตุการณ์ | ผล |
|-----------|-----|
| ขอท้า | จ่าย**ค่าหัว** → เข้าเจ้าของอันดับ (และ/หรือกองทุนโลก) |
| แพ้ | เสียค่าหัว · เจ้าของได้เงิน/soft fame |
| ชนะ | ได้ที่นั่งโชว์บนบอร์ด (ชื่อ·แรงก์·ฉายา) · เจ้าของเดิมหลุด |
| จำกัด | cooldown / จำกัดครั้งต่อวัน / ท้าได้เฉพาะช่วงอันดับใกล้เคียง (กันสแปม) |

**ไม่** ผูกชนะท้า = ลบเซฟหรือยึดของในตัวคนแพ้ — แค่แย่ง**บัลลังก์โชว์** + เงินค่าหัว

## 4. แผนเฟส implement (W0–W4)

**ยังไม่เริ่ม** จนกว่า single-player + เศรษฐกิจโลก + playtest บาลานซ์จะนิ่งพอ

| เฟส | ชื่อ | ขอบเขต | ความรู้สึก “เซิร์ฟ” | เงื่อนไขเริ่มโดยประมาณ |
|-----|------|--------|---------------------|-------------------------|
| **W0** | Rank board soft | บอร์ดจาก saves · soft band · help title · `rank_board.json` · score ซ่อน | ต่ำ | **1.13.15** (ฐาน) |
| **W1** | Player echo | เงาจากเซฟอื่นในโลก · สู้/คุยด้วย combat/social snapshot | กลาง (offline) | W0 นิ่ง |
| **W2** | Rank challenge | ท้าอันดับ + ค่าหัว + top N + cooldown | กลาง–สูง | W1 + เศรษฐกิจเงินพอ |
| **W3** | World host process | process โลกรันค้าง · client ต่อ local/remote | สูง | หลัง W2 + concurrent ออกแบบแล้ว |
| **W4** | Online lite | auth เบา · anti-cheat เบา · (optional) live presence | สูงสุด | product พร้อม beta |

### W0 — Rank board soft (รายละเอียด)

- อ่านเซฟใน `saves/{world_id}/`
- คำนวณ score **ซ่อน** (ไม่ dump ลง UI)
- โชว์ top N: ชื่อ · ฉายา · soft rank label
- เงื่อนไขติดบอร์ดซ่อน (คล้าย special mission)

### W1 — Player echo (รายละเอียด)

- สร้าง **combat snapshot** ตอนเซฟ/ออกสนาม: สกิลที่ใช้ได้, เกียร์หลัก, สเตตัสลงทุน, passive ปาร์ตี้, rarity soft
- เจอใน sights: kind `player_echo`
- สู้ = AI ใช้ snapshot · ผลไม่เขียนกลับเซฟเจ้าของ
- คุย/ทีม = personality + assist จาก snapshot · ลูทเป็นของ live player เท่านั้น
- Sanitize: ตัด admin flag, cap ค่าผิดปกติ

### W2 — Rank challenge (รายละเอียด)

- ขอท้าอันดับ K → จ่าย bounty
- แพ้/ชนะตามกฎ §3.4
- อัปเดต `rank_board.json` ต่อโลก
- soft text เงื่อนไขขึ้นบอร์ดยังไม่สปอยล์

### W3 — World host (รายละเอียด)

- แยก world service / lock ไฟล์หรือ queue เขียน
- market + rank + player index concurrent-safe
- terminal client ชี้ host

### W4 — Online lite (รายละเอียด)

- บัญชี/โทเคนเบา
- ตรวจ snapshot จาก server ไม่เชื่อ client เต็มที่
- (optional) แสดง soft “ร่องรอยสด” ถ้า online

## 5. โครงข้อมูลคร่าวๆ (เป้าหมาย)

```text
saves/{world_id}/
  {player_id}.json          # เซฟตัวละคร (ของเดิม)
  market.json               # ของเดิม
  rank_board.json           # ใหม่ (W0+) — public cards + hidden scores server-side
  echoes/{player_id}.json   # ใหม่ (W1+) — combat/social snapshot
  world_meta.json           # ใหม่ optional — รุ่น schema, flags
```

**Combat snapshot (แนว field ขั้นต่ำ)**

- level, occupation_id, skills[], equip summary, stats_alloc soft, party passives  
- **ไม่** ต้องทั้ง inventory ทุกชิ้น  
- version ติด snapshot สำหรับ migrate
