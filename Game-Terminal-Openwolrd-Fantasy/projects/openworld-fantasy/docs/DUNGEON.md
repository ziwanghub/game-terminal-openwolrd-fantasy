## 0. เป้าหมายผู้เล่น (Player fantasy)

| อยากได้ | ความหมายในเกม |
|---------|----------------|
| ลงลึกแล้วโหดขึ้น | มอน Lv / สเตต / AI / ดรอป สเกลตาม **ความลึกจริง** |
| บอสประจำชั้น | ต้องชนะบอสชั้นนี้ ถึงจะลงชั้นถัดไปได้ |
| ไม่รู้ลึกแค่ไหน | UI **ไม่โชว์** `1/N` — เห็นแค่ “ชั้นที่ลงมาแล้ว” + ความรู้สึกกดดัน |
| ออกได้ทุกเมื่อ | สำรวจ/ไฟต์มอนธรรมดา → เดินออกฟรี |
| ท้าบอส = ตัดใจ | บอส **หนีปกติไม่ได้** · ชนะ หรือใช้ **เศษหนี** (item) |
| ต้องมั่นใจก่อนท้า | ยืนยันชัด + soft ใบ้ความเสี่ยง · เช็คของหนีในมือ |

## 1. ปัจจุบัน (v1) vs เป้าหมาย (v2)

| หัวข้อ | v1 ตอนนี้ | v2 ที่ออกแบบ |
|--------|-----------|---------------|
| จำนวนชั้น | รู้ชัด `ชั้น 1/2` จาก data | **ซ่อนเพดาน** — แสดงแค่ “ลงมาชั้นที่ k” |
| เงื่อนไขลงลึก | สู้/เส้นทางพอ (ไม่บังคับบอส) | **ต้องกำจัดบอสประจำชั้น** |
| บอส | บอสตัวเดียวที่ชั้นในสุด | **บอสทุกชั้น** + บอสปลายทาง soft |
| สเกลมอน | ×hp/atk ตามดัน + ชั้นเล็กน้อย | ×**depth** ชัด: Lv · HP · ATK · ดรอป · XP |
| ทางออก | **ล็อก** จนเคลียร์/ของหนีตอนเข้า | **เปิดเสมอ** ยกเว้นระหว่างไฟต์บอส |
| หนีไฟต์ | หนีปกติได้ทุกไฟต์ | มอนทั่วไปหนีได้ · **บอสชั้น หนีปกติไม่ได้** |
| ของหนี | จำของตอนเข้าทั้งรอบ | ใช้จากกระเป๋า**ตอนนั้น** (เศษ) · ใช้ได้ในไฟต์บอส |
| เวลาภายใน | เทิร์นงบประมาณ ยุบมิติ | **ทางเลือก** — ลดความสำคัญหรือตัดออก (ขัด “ออกได้ทุกเมื่อ”) |

## 2. โครงหลัก v2

```
[สนาม] เจอปากดัน → เข้า (เตรียมปาร์ตี้ได้)
        ↓
[ในดัน · โหมดสำรวจ]
  · สำรวจชั้น → มอน/หีบ/กับดัก (สเกลตาม depth)
  · ท้าทายบอสชั้นนี้  ← ต้องยืนยัน
  · เดินออกจากดัน     ← ฟรี (เก็บของที่เก็บแล้ว)
  · ปาร์ตี้ / สถานะ / ขอแรง (ถ้าเปิด H*)
        ↓ ท้าบอส
[ไฟต์บอสชั้น]  หนีปกติปิด · ใช้เศษหนีได้ · แพ้ = soft eject
        ↓ ชนะ
  · ปลดล็อก “ลงลึก”
  · ดรอปชั้น / ชำนาญ
        ↓ ลงลึก
  depth += 1 · ภูมิชั้นใหม่ · มอนโหดขึ้น · บอสชั้นใหม่
        ↓ (ลึกมาก / เงื่อนไขปลาย)
  soft “หัวใจดัน” หรือจบรอบด้วยการเลือกออก
```

### 2.1 Depth (ความลึก) — แหล่งความจริงเดียว

| ฟิลด์รัน | ความหมาย | โชว์ผู้เล่น |
|----------|----------|-------------|
| `depth` | ชั้นปัจจุบัน เริ่ม 1 | soft: “ชั้นที่ลงมา” / ไม่มี `/max` |
| `max_depth_hidden` | เพดานรัน (seed ตอนเข้า) | **ไม่โชว์** |
| `floor_boss_cleared` | เคลียร์บอสชั้นนี้หรือยัง | soft: ทางลึกเปิด/ปิด |
| `boss_locked` | กำลังไฟต์บอส | เมนูสำรวจปิดชั่วคราว |
| `exits_allowed` | ออกได้ไหม (false ตอนบอส) | — |

**เพดานซ่อน (ตัวอย่างสูตร — ซ่อนจาก UI):**

```
max_depth = clamp(
  base_floors[dungeon] + roll(0..bonus) + knowledge_soft_bonus,
  min_floors, hard_cap
)
```

- ดันอ่อน (ป่า): base 2–4, hard_cap ~6  
- ดันโหด (สุญญะ): base 3–5, hard_cap ~9  
- ความรู้หลังเคลียร์หลายครั้ง → soft label “ลึกกว่าที่คิด” ไม่โชว์ตัวเลข

เมื่อ `depth == max_depth_hidden` และเคลียร์บอสชั้นนั้น → ถือว่า **เคลียร์ดันรอบนี้** (รางวัลใหญ่) แล้วทางลงลึกปิดด้วย soft “มิติดันจบที่นี่ — ทางออกเปิดโล่ง”

### 2.2 บอสประจำชั้น

| ชนิด | เมื่อไหร่ | หนีปกติ | ดรอป |
|------|----------|----------|------|
| **Floor boss** | ท้าทายหลังสำรวจ/พร้อม | ❌ ปิด | ชั้น-scaled + โอกาสเศษหนี |
| **Heart boss** (optional) | ชั้นสุดท้ายซ่อน | ❌ ปิด | รางวัลดัน + knowledge clear |

**เงื่อนไขลงลึก**

1. `floor_boss_cleared == true`  
2. ไม่ได้อยู่ในไฟต์  
3. soft: อาจต้อง “สำรวจชั้นอย่างน้อย 1 ครั้ง” ก่อนท้าบอส (กันบอสทันทีตอนเข้า — ปรับบาลานซ์ได้)

**สปอว์นบอสชั้น**

- ดึงจาก pool ตาม `area_id` + tag ดัน  
- สเกลด้วย `depth` เหมือนมอน (ดู §3)  
- ชื่อ soft: “ผู้เฝ้าชั้น” / “เงาที่ขวางบันได” จน knowledge พอ  
- ชั้นลึกมาก → โอกาส elite/phase boss สูงขึ้น (ไม่บอกสูตร)

### 2.3 ทางออก — ออกได้ทุกเมื่อ (ยกเว้นบอส)

| สถานะ | ออกจากดัน | หนีไฟต์ |
|--------|-----------|---------|
| สำรวจชั้น | ✅ ฟรี (ไม่ drain หรือ drain เบา optional) | — |
| ไฟต์มอนทั่วไป | — | ✅ หนีปกติได้ (กฎสนามเดิม) |
| ยืนยันท้าบอสแล้ว / ไฟต์บอส | ❌ ออกดันไม่ได้จนกว่าจบไฟต์ | ❌ หนีปกติ · ✅ **เศษหนี** |
| หลังบอสแพ้ (soft death) | ถูกเหวี่ยงออก + drain | — |
| หลังบอสชนะ | ✅ ออกได้ · ✅ ลงลึกได้ | — |

**ยืนยันก่อนท้าบอส (บังคับ UX)**

```
ท้าทายผู้เฝ้าชั้น?
 · หนีปกติใช้ไม่ได้
 · ชนะ = ทางลงเปิด · แพ้ = ถูกเหวี่ยงออก (เสียของบางส่วน)
 · เศษหนีในมือ: มี N ชิ้น / ไม่มี
  1  มั่นใจ — ท้า
  0  ยังไม่พร้อม
```

## 3. สเกลตามความลึก (มอน · ของ · XP)

ตัวเลขเป็น **ภายใน** — UI ใช้ soft band เท่านั้น

### 3.1 มอนสเตอร์ (explore combat)

```
lv_bonus   = floor((depth - 1) * lv_per_depth)          # เช่น 1–2 lv ต่อชั้น
hp_mult    = 1 + (depth - 1) * 0.10 + dungeon_bias
atk_mult   = 1 + (depth - 1) * 0.08 + dungeon_bias
ai_tier    = soft_band(depth)   # โง่ → ระวัง → ฉลาด (MI)
rarity_bias= +depth soft        # โอกาส uncommon+ สูงขึ้นเล็กน้อย
```

### 3.2 บอสชั้น

```
boss_hp  ≈ normal * (1.6 + 0.15 * depth) * dungeon_bias
boss_atk ≈ normal * (1.3 + 0.10 * depth) * dungeon_bias
phases   = 1 + (depth >= 4) + (depth >= 7)   # soft ซ่อน
```

### 3.3 ดรอป / XP / เงิน

| แหล่ง | สเกล depth |
|--------|------------|
| XP มอน | `× (1 + 0.12*(depth-1))` |
| เงิน | `× (1 + 0.10*(depth-1))` |
| คุณภาพดรอป | chance rare+ ขยับตาม depth (cap) |
| หีบชั้น | ใช้ source `dungeon_floor` + depth tier |
| บอสชั้น | source `dungeon_floor_boss` |
| เคลียร์ดัน | source `dungeon_clear` (ครั้ง/รอบ) |

**เศษหนี (escape shard)** — ดรอปเบาจากบอสชั้น / หีบลึก / คราฟ (ไม่สอนสูตรเต็ม)

## 4. ของหนี / เศษหนี (Boss-only escape)

### 4.1 บทบาท

| โหมด | ของหนี |
|------|--------|
| นอกไฟต์บอส | **ไม่จำเป็น** — เดินออกได้ |
| ในไฟต์บอส | ทางเดียวที่ “ถอย” ได้โดยไม่แพ้เต็มรูป |

### 4.2 พฤติกรรมในไฟต์บอส

1. เมนูไฟต์: **หนี** → ข้อความ “วงบอสขังคุณ — หนีธรรมดาไม่ได้”  
2. เมนูใหม่ / sub: **ใช้เศษหนี** (ถ้ามีในกระเป๋า)  
3. ใช้แล้ว: โอกาสสำเร็จตามของ · สำเร็จ = ออกจากไฟต์บอส + ยังอยู่ในดัน (หรือเหวี่ยงออก soft — เลือกบาลานซ์)  
4. แนะนำดีไซน์: **สำเร็จ = กลับโหมดสำรวจชั้นเดิม** (บอสยังอยู่ ยังไม่เคลียร์) + ของถูกกิน · เสีย HP/แรงกดเล็กน้อย  
5. ล้มเหลว = เสียของ + โดนโจมตีฟรี / กดดันพุ่ง

### 4.3 รายการของ (ขยายจาก v1)

| id | โทน | ใช้ตอนบอส | หมายเหตุ |
|----|-----|------------|----------|
| `dungeon_thread` | ด้ายกลับทาง | สำเร็จปานกลาง | ดรอป/ร้านหายาก |
| `void_key_shard` | เศษกุญแจ | สำเร็จสูง | ดัน void / ลึก |
| `blessed_charm` | เครื่องราง | สำเร็จต่ำ | multi-use soft |
| `escape_shard` (ใหม่) | เศษหนีทั่วไป | ปานกลาง | ดรอปบอสชั้น |

v1 “ต้องมีตอนเข้า” **เลิกใช้เป็นเงื่อนไขหลัก** — ใช้ของในกระเป๋าตอนกดใช้

## 5. ลูปการเล่น & เมนู

### 5.1 เมนูในดัน (สำรวจ)

```
ทำอะไรในดัน
  1  สำรวจชั้นนี้     (มอน/ของ/กับดัก · หนีไฟต์ได้)
  2  ท้าทายบอสชั้น   (ยืนยัน · หนีปกติไม่ได้)
  3  ลงลึกกว่านี้     (ต้องเคลียร์บอสชั้น · ถ้าถึงเพดานซ่อน → จบรอบ)
  4  เดินออกจากดัน   (ฟรี · เก็บของที่เก็บแล้ว)
  5  ดูสถานะดัน
  6  สัญญาณขอแรง
  Y  ปาร์ตี้
```

### 5.2 แผงสถานะดัน (soft)

```
ดันเจียน · รากป่ารูปเขี้ยว
  ความลึก   ลงมาชั้นที่ 3        ← ไม่มี /max
  ภูมิชั้น   …
  ผู้เฝ้า    ยังขวางทาง / ทางลงเปิดแล้ว
  สัญญาณ   ยิ่งลึก ยิ่งหนัก (knowledge)
  ทางออก   เปิด (สำรวจ) · ปิดเฉพาะตอนไฟต์บอส
```

### 5.3 ความรู้ (anti-spoiler)

- ครั้งแรก: ไม่รู้ลึกแค่ไหน · soft “โพรงมืดไม่รู้จบ”  
- เคยลงลึก k: “เคยถึงระดับหนึ่ง…”  
- เคลียร์: soft accurate ตามดัน  
- **ห้าม** โชว์ difficulty number / max floor / % ดรอป

## 6. ความล้มเหลว & ความเสี่ยง

| เหตุ | ผล |
|------|-----|
| สลบไฟต์มอนทั่วไป | soft death ในดัน หรือเหวี่ยงออก + drain เบา |
| สลบไฟต์บอสชั้น | เหวี่ยงออก + drain แรงกว่า + บอสชั้นไม่เคลียร์ |
| ใช้เศษหนีสำเร็จ | กลับสำรวจ / หรือออก (ตามของ) · บอสไม่เคลียร์ |
| เดินออกเอง | ไม่ drain หรือ drain น้อยมาก · เก็บ loot แล้ว |
| ถึงเพดาน + เคลียร์บอสสุด | รางวัลเคลียร์ดัน + knowledge clear |

**เวลาภายใน v1:** ขัดกับ “ออกได้ทุกเมื่อ” → v2 **ปิดเป็นค่าเริ่ม** หรือเหลือแค่ soft flavor อากาศ (ไม่ยุบบังคับ)

## 7. Data shape (เป้าหมาย)

```yaml
# dungeons.yaml (v2 fields)
dungeons:
  - id: dung_forest_root
    name: รากป่ารูปเขี้ยว
    area_id: dark_forest
    difficulty: 2              # hidden bias only
    depth:
      base_min: 2
      base_max: 4
      hard_cap: 6
      lv_per_depth: 1.5
    enemy_bias:
      hp: 1.15
      atk: 1.10
    floor_boss_pool: [elite_forest_*, boss_forest_warden]
    heart_boss_id: boss_forest_king
    rewards_clear: { ... }
    soft_labels: { vague, known, accurate }
```

```python
# dungeon_run (runtime)
{
  "dungeon_id": "...",

  "boss_encounter_active": False,
  "layout": {...},
  "path_progress": 0,
  # เลิกพึ่ง locked ตลอดรอบ
```
