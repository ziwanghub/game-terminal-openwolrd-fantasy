## 1. ภาพรวมเฟส (IC0 → IC6)

```text
IC0  ล็อกกฎ + map ชนิด          ← เอกสาร / ไม่พังเกม
  │
  ▼
IC1  เกียร์ช่องบาง + ชนิดอาวุธ   ← โหลดเอาต์เล่นได้หลาก
  │
  ▼
IC2  การ์ดผูกมอน (ชุด 1)        ← ลูปล่าการ์ดชัด
  │
  ▼
IC3  Mat + คราฟสายมอน           ← ล่า→คราฟ
  │
  ▼
IC4  Mid–late เกียร์/การ์ด/บอส  ← ความลึก mid game
  │
  ▼
IC5  หีบ pool + ร้าน + เซ็ต      ← เศรษฐกิจปิดวง
  │
  ▼
IC6  บาลานซ์ทั้งก้อน + polish     ← “อัปครบ” นิ่งพร้อม playtest ยาว
```

| เฟส | ชื่อสั้น | เป้าจำนวน (สะสมโดยประมาณ) | เกณฑ์จบเฟส |
|-----|----------|---------------------------|------------|
| **IC0** | กฎ + map | เอกสาร | ทีมล็อกกฎ rename/map/compatible |
| **IC1** | เกียร์ pack A/B | ไอเทม ~115 · เกียร์ ~50+ | ทุกช่องมีชั้น · อาวุธหลายชนิด · **1.28–1.29** |
| **IC2** | การ์ด pack A | การ์ด **~36→46** | มอน rebind + MC3 cards · **1.30–1.31** |
| **IC3** | Mat/คราฟ | คราฟ ~46+ · mat สายมอน | ล่า→คราฟ · **1.31 done** |
| **IC4** | Mid–late | เกียร์บอส+การ์ด+เซ็ต | **1.32 done** |
| **IC5** | เศรษฐกิจปิด | หีบ/ร้าน mid · คราฟ late | **1.33 done** |
| **IC6** | polish ทั้งก้อน | sink · หีบ bias · คราฟเบา · health tests | **1.34 done** |

**หมายเหตุ:** เลขสะสมยืดหยุ่น — ใช้เป็นเข็มทิศ ไม่ใช่โควตาแข็ง

## 2. รายละเอียดรายเฟส

### IC0 — กฎและแผนที่ชนิด (ไม่ต้อง bulk data)

| งาน | รายละเอียด |
|-----|------------|
| Map ช่อง RO-like → EL | weapon→main_hand · shield→off_hand · head→head · armor→body · shoes→feet · acc→acc_1 · garment→body เบา/ข้าม |
| Map ชนิดอาวุธ | ดาบ · มีด · หอก · ขวาน · กระบอง · staff · 2H · (ธนู soft ทีหลัง) |
| กฎการ์ด | 1 มอนหลัก : 1 การ์ด · ร้านไม่ขาย · คราฟการ์ดมอนไม่ได้ · rate soft |
| กฎชื่อ | ภาษาไทย/โลกเรา · ห้ามชื่อ RO ต้นฉบับเป็น id/name |
| กฎโบนัส | common แผ่ว · rare/บอสชัดขึ้น · diminishing ถ้าซ้อน |

**ส่งมอบ:** เอกสารนี้ + ตาราง map ใน §4  
**สถานะ:** โครงพร้อม (บางส่วนทำแล้วใน 1.24–1.27)

---

### IC1 — เกียร์ pack A (ช่องบาง + ชนิดอาวุธ)

| งาน | ตัวอย่าง |
|-----|----------|
| เติม **head / legs / feet** | อย่างน้อย 1–2 ชิ้น/ช่อง ชั้น uncommon–rare |
| เติม **off_hand** | โล่ mid · focus ชั้น 2 |
| เติม **อาวุธชนิด** | หอก/ขวาน/กระบอง ชั้นถัดไป (มีพื้นฐานแล้ว) |
| ผูก **weight_class / material** | ให้ EQ soft ทำงานครบขึ้น |
| ร้าน armory | สต็อกชิ้นใหม่ชั้น early–mid |

**อย่าทำใน IC1:** dump เกียร์ 100 ชิ้น · legendary ทั้งแผ่น

**เกณฑ์จบ:** สวม 7 ช่องแล้วรู้สึกมีทางเลือก · pytest data integrity ผ่าน

---

### IC2 — การ์ด pack A (ผูกมอน)

| งาน | รายละเอียด |
|-----|------------|
| การ์ดใหม่ ~15–25 ใบ | ผูกมอนที่ยังใช้การ์ดธาตุซ้ำ |
| `compatible` เป็นช่อง EL | `main_hand` `body` `head` `feet` `off_hand` `acc_1` `legs` |
| `card_id` / `card_rate` บนมอน | 1 ใบต่อมอนหลัก (บอสแยก rate) |
| โบนัส soft | HP/ATK/on_hit/status แผ่ว — เทสไฟต์ |

**อย่าทำใน IC2:** card compound RO · set โหด · โชว์ %

**เกณฑ์จบ:** ฆ่ามอนแล้วโอกาสได้การ์ด “ของมอนนั้น” · socket ได้ · ร้านยังไม่ขายการ์ด

---

### IC3 — วัตถุดิบ + คราฟสายมอน

| งาน | รายละเอียด |
|-----|------------|
| Mat เฉพาะพื้นที่ที่ยังบาง | เมือง/ผลึก/รอยแยก เสริมถ้าขาด |
| คราฟ forge/camp/mystic | mat มอน → เกียร์/ยา/ชา (K3 station คงไว้) |
| เควส soft 1–2 | “รวบรวมเขี้ยว/เศษหิน” แบบ soft |

**เกณฑ์จบ:** อย่างน้อย 3 พื้นที่ ล่า→คราฟ รู้เรื่องโดยไม่เปิด wiki

---

### IC4 — Mid–late (เกียร์ + การ์ด + บอส)

| งาน | รายละเอียด |
|-----|------------|
| เกียร์ rare–sacred ต่อช่อง | ไม่การันตีจากหีบอย่างเดียว — มีคราฟ/บอส |
| การ์ดบอส / พื้นที่สูง | rate ดีกว่ามอนธรรมดา ยัง soft |
| อาวุธชนิดลึก | dual-specialty · 2H ชั้นถัด · focus สูง |
| unit unique คงเดิม | **ไม่คราฟ** · หีบ unit เท่านั้น |

**เกณฑ์จบ:** mid game มีเป้าหมายของ · บอสรู้สึกคุ้ม

---

### IC5 — ปิดวงเศรษฐกิจ (หีบ · ร้าน · เซ็ต)

| งาน | รายละเอียด |
|-----|------------|
| หีบ pool | อ้าง item id ใหม่ใน buckets |
| ร้าน | ขายเกียร์/mat ชั้นที่กำหนด — **ไม่ขายการ์ด · ไม่ขาย unit/หีบแรง** |
| เซ็ตเกียร์/การ์ดเบา | 2–4 สาย (ขยายจาก iron/shadow/arcane/thorn) |
| ตลาด P2P | รองรับ id ใหม่โดยอัตโนมัติถ้าใช้ registry |

**เกณฑ์จบ:** ของใหม่ทุกชิ้นมีอย่างน้อย 1 แหล่งได้ (ดรอป/คราฟ/ร้าน/หีบ/เควส)

---

### IC6 — บาลานซ์ทั้งก้อน + ปิดงาน “อัปครบ”

| งาน | รายละเอียด |
|-----|------------|
| จูน rate ดรอปทุกพื้นที่ | จาก playtest (แน่น/โล่ง) |
| จูนโบนัสการ์ด/เกียร์ | ไม่พลิก ATB/needs |
| เอกสารแคตตาล็อก | สรุปจำนวน · ตารางชนิด · changelog |
| เทส regression | data integrity · craft · loot pick · shop no cards |

**เกณฑ์จบแผน IC:** ครบเป้าปลายทาง · solo playtest 2–4 ชม. ไม่มี blocker ไอเทม · **พร้อมโฟกัสอย่างอื่น (หรือ online ทีหลังตาม ROADMAP)**

## 3. สิ่งที่ “อัปครบ” ครอบคลุม (checklist ปลายทาง)

- [ ] เกียร์ครบ 7 ช่อง · หลาย tier · หลายชนิดอาวุธ

- [ ] ร้าน/หีบ/เควส อ้าง id ใหม่  
- [ ] เซ็ตเบา 2–4 สาย  
- [ ] ไม่มีชื่อ/ศิลป์ RO ต้นฉบับใน data  
- [ ] Soft rules คง · เทสเขียว

## 5. กฎต่อเฟส (คุณภาพ)

ทุกเฟส IC1–IC6 ต้อง:

1. Data ใน `data/items` / `data/cards` / `data/monsters` / `data/craft` / `data/shops` / `data/chests` ตาม scope  
2. `pytest` ผ่าน (อย่างน้อย unit + data_validation)  
3. ไม่พังเซฟเก่า (id ใหม่เท่านั้น · ไม่ลบ id เก่าโดยไม่ migrate)  
4. อัป `PHASES.md` + บรรทัดสถานะใน `IMPROVEMENT_PLAN.md`  
5. Playtest สั้นหลังเฟสใหญ่ (IC2, IC4, IC6)
