## 2. โมเดลค่า (ภายใน)

เก็บ `player["needs"]` แบบขยาย:

```text
hunger:   -50 … +100   # >0 หิวสะสม · <0 “อิ่มเกิน/สำรอง” (บัพเบา)
fatigue:  -50 … +100   # >0 ล้า · <0 สดชื่นเกิน
morale:   -50 … +100   # >0 ขวัญดี · <0 ขวัญเสีย (ติดลบ = แย่)
```

**เทียบ T0 ปัจจุบัน (0–100 หิวสูง=แย่):**  
เฟส N1 จะ **migrate** เป็นสเกลใหม่:

| สเกลใหม่ | ความหมาย UI |
|----------|-------------|
| hunger ≤ 0 | อิ่ม / สำรอง (แถบเต็ม + ไม่มี −) |
| hunger 1–40 | ปกติ–เริ่มหิว |
| hunger 41–70 | หิว |
| hunger 71–90 | หิวมาก |
| hunger > 90 หรือ “ติดลบขวัญรวม” | วิกฤต · เสี่ยง soft death |

**ทางเลือก implement ที่ง่ายกว่า (แนะนำ):**  
คง 0–100 ภายใน แต่เพิ่ม `strain` แยก:

```text
# แนะนำใช้แบบนี้ในโค้ดจริง — ไม่ง่ายพัง T0
hunger 0–100   # 0=อิ่ม 100=อด
fatigue 0–100
morale 0–100   # 0=ย่ำแย่ 100=ดี

# UI “ติดลบ” = เมื่อ band == crit แสดงเครื่องหมาย − และแถบแดง/สั้นจัด
# ไม่ต้องใช้ค่าติดลบในตัวเลขจริง
```

**UI ที่คุณเสนอ (เครื่องหมาย −):**

```text
กรณีปกติ:
  ท้อง █████░░░   ล้า ███████░   ขวัญ ████████

กรณีไม่ปกติ (หิว / ขวัญตก):
  ท้อง − ██░░░░░░   ล้า ███████░   ขวัญ − ███░░░░░
         ↑ เครื่องหมายลบ = ภาวะแย่ · แถบสั้นลงเรื่อยๆ
```

- **−** โชว์เมื่อ band ≥ `bad` (หิว/ล้า) หรือ morale band ≤ `low`  
- แถบยิ่งสั้น = ยิ่งแย่ (คง logic T0: เต็ม = สบาย สำหรับท้อง/ล้า)  
- เมื่อเข้า **crit** แถบอาจแสดงเป็น `−− ░░░░░░░░` + ข้อความ soft

**ไม่** แสดงตัวเลข −37 ให้ผู้เล่น — แค่สัญลักษณ์ภาวะ

## 3. ผลต่อพลังรบ (Combat modifiers)

คำนวณ **multipliers ซ่อน** ตอนเข้าไฟต์ / ตอน resolve ดาเมจ (ไม่ dump UI):

### 3.1 ท้อง (hunger) → โจมตี · ป้องกัน · หลบ

| band หิว | ATK | DEF (รับดาเมจ) | Dodge |
|----------|-----|----------------|-------|
| อิ่ม (good) | ×1.00 ~ ×1.02 | ×1.00 | ×1.00 |
| ปกติ (mid) | ×1.00 | ×1.00 | ×1.00 |
| หิว (bad) | ×0.92 | รับดาเมจ ×1.06 | ×0.90 |
| อด (crit) | ×0.80 | รับดาเมจ ×1.14 | ×0.75 |

- หิวมาก → ตีอ่อน · แทงก์แย่ · หลบช้า  
- อิ่มดีมาก (อาหารดี) อาจ ATK/DEF +soft ชั่วคราวจากบัฟอาหาร ไม่จาก hunger ติดลบถาวร

### 3.2 ล้า (fatigue) → ความเร็วแท่ง ATB

ผูก `combat_atb` fill rate:

| band ล้า | ATB fill mult | ความหมาย |
|----------|--------------|----------|
| เบา (good) | ×1.05 | แท่งเต็มเร็วเล็กน้อย |
| ปกติ | ×1.00 | ปกติ |
| ล้า (bad) | ×0.88 | แท่งช้า · ศัตรูมีโอกาสก่อน/บ่อยกว่า |
| หมดแรง (crit) | ×0.72 | ช้าชัด · ถูกกดดัน |

สูตรแนว:

```text
atb_rate *= fatigue_mult(player)
atb_rate *= (1 + f(speed_alloc, power_spd))   # ของเดิม
```

**ล้าเต็มแถบ (สด) = เร็วปกติ/เร็วเล็กน้อย**  
**ล้าแถบสั้น = ช้า** — ตรงที่คุณอธิบาย

### 3.3 ขวัญ (morale) → โอกาสสำเร็จ · สกิล · บอส

| band ขวัญ | ผล |
|-----------|-----|
| ขวัญดี | skill_fail ×0.5 · crit soft +เล็ก · เข้าบอสไม่สั่น |
| มั่นคง | ปกติ |
| หด | skill_fail +8% soft · บางสกิล “มือสั่น” (พลาด) |
| ย่ำแย่ | skill_fail +18% · สกิลบางแท็ก `requires_focus` ใช้ไม่ได้หรือบังคับพลาด · เจอบอส: ข้อความขวัญตก + โอกาสพลาดคอมโบ |

**สกิลพลาด:** ไม่โชว์ % — ข้อความ soft  
`「จังหวะหลุด… ขวัญยังไม่นิ่ง」`

**บอส:** ก่อนเข้า ถ้า morale crit → ใบ้  
`「เงาใหญ่ทำให้ขาอ่อน — อาจยังไม่พร้อม」`  
ไม่บล็อกตายตัว (open world) แต่โทษในไฟต์ชัด

### 3.4 หิวจัด → เสี่ยงตาย (soft)

เมื่อ `hunger` band = crit **และ** ทำ action หนัก (สำรวจ/ไฟต์/เดินทาง) roll ซ่อน:

```text
collapse_chance = base(hunger) * (1 - resist_from_def_hp)
```

ผล = **soft death** (เหมือนแพ้): HP เหลือเศษ · เสียเงิน/ของเบา · ข้อความ  
`「ร่างกายไม่ไหว… สลบเพราะความหิว」`  
**ไม่** hard delete เซฟ

## 4. ผูกกับแต้มสเตตัส (ผกผัน / ต้านทาน) — ซ่อนสูตร

ผู้เล่นลงทุนใน P เมนู: **ATK · DEF · MAG · SPD · INT**

| สเตตัส | ผลต่อ needs (ซ่อน) |
|--------|---------------------|
| **defense** (+ HP bias) | ล้าเพิ่มช้า · หิวจัด soft death ยากขึ้น · ทน penalty DEF จากหิว |
| **speed** | ชะลอโทษ ATB จากล้า (ล้าแล้วยังไม่ช้าเท่าคน SPD ต่ำ) · หลบโดนหิวน้อยลง |
| **intelligence** (+ สติ) | ขวัญตกช้า · skill_fail จากขวัญต่ำลด · สมาธิ: ฟื้น morale ตอนพักดีขึ้น |
| **atk** | ต้าน ATK-penalty จากหิวเล็กน้อย (ยังตีได้แม้หิว — “ความดื้อ”) |
| **magic** | คล้าย INT เบาสำหรับสกิลเวท fail |

แนวสูตร (ไม่โชว์):

```text
fatigue_gain *= 1 / (1 + 0.03 * def_pts + 0.01 * power_def)
morale_loss  *= 1 / (1 + 0.04 * int_pts + 0.02 * power_intel)
atb_fatigue_penalty *= 1 / (1 + 0.035 * spd_pts)
hunger_atk_penalty  *= 1 / (1 + 0.02 * atk_pts)
```

**ความหมายเล่น:**  
tank/DEF ล้าช้า · นักเวท/INT ขวัญนิ่ง · SPD ยังมีจังหวะแม้ล้า · ATK หิวยังตีได้บ้าง

**Pressure / สติ (intel_current):**  
สติต่ำ + ขวัญต่ำ → fail ซ้อน  
สติสูง → ลด skill_fail จาก morale

## 5. ระบบอาหาร (Items · ร้าน · ระดับ · บัฟ)

### 5.1 แยกจากยา

| ชนิด | บทบาท |
|------|--------|
| **potion / ยา** | HP MP สถานะ — **ไม่** ควรเป็นทางหลักลดหิว (หรือลดหิวแค่นิด) |
| **food / เสบียง** | ลด hunger เป็นหลัก + อาจบัฟชั่วคราว |
| **luxury food** | แพง · บัฟแรงสั้น · ขวัญขึ้นชัด |

แท็ก data:

```yaml
- id: city_bread
  kind: consumable
  tags: [food]
  food_tier: 1          # 1=หยาบ … 5=เลิศ
  hunger_relief: 35     # ลด hunger ภายใน
  fatigue_relief: 5
  morale_boost: 4
  price_world: 12
  # optional buff
  food_buff:
    id: warm_meal
    duration_actions: 8
    atk_mult: 1.03
```

### 5.2 ระดับอาหาร (food_tier)

| tier | ชื่อ soft | ราคาแนว | hunger_relief | บัฟ |
|------|-----------|---------|---------------|-----|
| 1 | หยาบ / แห้ง | ถูก | 25–35 | ไม่มีหรือขวัญ + |
| 2 | บ้านๆ | ปานกลาง | 40–50 | ATK หรือ DEF เบา |
| 3 | ดี | แพงขึ้น | 55–65 | ATK+DEF หรือ SPD เบา |
| 4 | เลิศ | แพง | 70–80 | บัฟชัด + ขวัญ |
| 5 | ตำนาน/พิธี | แพงมาก / เงินพิเศษ | เต็ม + overfull | บัฟแรงสั้น · อาจ unlock flavor unit |

ขายคืนร้าน: `sell = buy * ratio * tier_factor` (tier สูงขายได้สัดส่วนดีกว่าเล็กน้อย)

### 5.3 ร้าน

| ร้าน | stock อาหาร |
|------|-------------|
| พ่อค้าเร่ | tier 1–2 (ration, bread) |
| ร้านเมือง/โรงเตี๊ยม (ใหม่หรือขยาย) | tier 1–3 |
| ตลาดสวรรค์ | tier 3–4 “สำรับศักดิ์” |
| ตลาดนรก/หายาก | tier 4–5 เสี่ยง/ของแปลก |

ทุกพื้นที่เมืองควรมี **อย่างน้อย 1 แหล่งอาหาร** (playtest feedback)

### 5.4 ใช้จากกระเป๋า

- เมนูใช้ของ: ถ้า `tags` มี `food` → `apply_needs_event` แบบ food (ไม่ใช้ยา)  
- ยา: ลด hunger น้อยมากหรือ 0  
- ข้อความ soft: `「อิ่มขึ้น…」` / `「อุ่นท้อง — แรงฮึด」` ถ้ามีบัฟ

### 5.5 บัฟอาหาร (status ชั่วคราว)

ใช้ `status_fx` ที่มีแล้ว:

| buff id | ผล soft |
|---------|---------|
| warm_meal | ATK เบา |
| hearty_stew | DEF / ล้าน้อยลงชั่วคราว |
| trail_mix | SPD / ATB เบา |
| feast_morale | morale สูง + skill fail ลด |

duration นับเป็น **actions สนาม/ไฟต์** ไม่โชว์วินาที

## 6. Unit / พลังเฉพาะ (ได้โดยไม่รู้วิธี)

สอดคล้อง occupation unit + anti-spoiler:

| แนว unlock (ซ่อน) | ตัวอย่าง soft ที่ผู้เล่นเห็น |
|-------------------|------------------------------|
| เคย soft death จากหิว ≥ 1 ครั้ง แล้วรอด | 「ร่างจำความอดอยาก」 |
| ช่วย H3 สำเร็จ + อาหาร tier≥3 | 「ผู้แบ่งปันเสบียง」 |
| morale crit แล้วยังชนะบอส | 「ใจไม่แตก」 |
| กิน food_tier≥4 ครบชุดเงื่อนไข | unit passive อาหาร |

**ผู้เล่นไม่เห็น checklist** — เห็นแค่ฉายา/unit โผล่  
passive ตัวอย่าง (ซ่อนตัวเลข):

- หิว penalty ครึ่งหนึ่ง  
- ATB ล้าช้าลงน้อยลง  
- เริ่มไฟต์บอส morale ไม่ต่ำกว่า mid  

เก็บ flag ใน `player["flags"]` / `unit_class_id` เหมือนระบบ unit เดิม
