## 0. เป้าหมายหนึ่งประโยค

ให้การแต่งตัว/ต่อสู้**หลากหลายและอ่านไม่ง่าย** — ผู้เล่นทดลอง สังเกต soft feedback  
**ไม่** เปิดตาราง “ใส่ X = +12% กันไฟ” แบบเกมที่ theorycraft ทันที

## 1. หลัก “คาดเดายาก” (Soft Theorycraft Resistance)

### 1.1 ปัญหาของเกมทั่วไป

| เกมทั่วไป | ผล |
|-----------|-----|
| สเตตัสดิบ + ตารางต้านโชว์ | รู้ทันทีว่าต้องสแตก def / ใส่เกียร์ธาตุอะไร |
| DPS meter / ตัวเลขคริชัด | ฟาร์ม build เดียว |
| ช่องสวมตายตัว + wiki | meta ตายตัวเร็ว |

### 1.2 หลักที่ล็อกในเกมนี้ (S-P1…S-P8)

| # | หลัก | ความหมายในระบบ |
|---|------|----------------|
| **S-P1** | Soft bars / soft labels | กัน/ธาตุ/คริ แสดง「รับได้」「แสงแผด」「คริ!」ไม่โชว์ % |
| **S-P2** | Multi-axis soft stack | ดาเมจ = ชั้นโจมตี × ธาตุ × กันสกิล × needs × โชค — **ไม่ใช่สเตตัสเดียว** |
| **S-P3** | Diminishing / soft cap ซ่อน | สแตก def สูงเกิน → ผลเพิ่มช้า (clamp เดิม ~55%) |
| **S-P4** | Context matter | พื้นที่ · mastery · needs · เวลากลางวัน soft — ชุดเดียวไม่ชนะทุกที่ |
| **S-P5** | Loadout trade-off | สองมือแรงแต่ไม่มีโล่ · คู่ดาบเร็วแต่กันอ่อน · ชุดหนักช้า ATB |
| **S-P6** | Hidden affinity | บางอาชีพ/นิสัย/unit “เข้ากับ” ช่อง/ธาตุ — ใบ้ soft ไม่โชว์ตัวคูณ |
| **S-P7** | Set / instance noise | เซ็ตทำงาน soft · rarity ชิ้น · instance id — ของ “ชื่อเดียวกัน” ไม่เท่ากันเสมอ |
| **S-P8** | Experiment loop | หีบ · คราฟเสี่ยง · ตลาด — ได้ของไม่คาด → ลองใส่ ไม่ใช่ farm checklist |
| **S-P9** | Weight class soft | เบา/กลาง/หนัก — trade-off ATB↔กัน↔needs ไม่โชว์ kg |
| **S-P10** | Grip affinity | ชินโหมดอาวุธ (โล่/dual/2H) ช้า · สลับบ่อยมีต้นทุน soft |
| **S-P11** | Occ/personality bias ไม่ล็อก | อาชีพ·นิสัยโน้มชุด — **ใส่ผิดอาชีพได้** แค่แผ่ว soft |
| **S-P12** | Area × material | วัสดุเกียร์ปะทะสภาพพื้นที่ — ชุดเดียวไม่ชนะทุกแมพ |
| **S-P13** | Partial set + mix | เซ็ตไม่ครบยังมีค่า · ปะวัสดุ = noise ไม่ใช่ bonus ตายตัว |
| **S-P14** | Situation lock | ในไฟต์/ดันลึก เปลี่ยนเกียร์ช้า/ไม่ได้ — ต้องเดาก่อนเข้า |

**ผลที่อยากได้:**  
ผู้เล่นถามว่า “ใส่ชุดไหนดี?” → คำตอบคือ **ลองในป่ามืด / ดัน / บอส แล้วดู soft**  
ไม่ใช่ “เปิด wiki แล้วใส่ครบ”

### 1.3 สิ่งที่**ยังโชว์ได้** (ไม่ทำให้งงเกิน)

| โชว์ | ไม่โชว์ |
|------|---------|
| ชื่อของ · ช่อง · soft rarity | ตัวเลข ATK ดิบหลังทุกสูตร |
| แถบ HP/MP · soft กัน strong/weak | ตาราง % ต้านธาตุ |
| 「ลงทุนแล้ว ×n」 | power_def / mdef ดิบ |
| ช่องว่าง / มือซ้าย-ขวา | น้ำหนัก meta exact |

## 3. โมเดลช่องใหม่ (ออกแบบ)

### 3.1 แผนที่ช่อง (Loadout map)

```text
        [ head ]
[ main_hand ] [ off_hand ]
        [ body ]
        [ legs ]
        [ feet ]
     [ accessory ×N ]
```

| ช่อง UI | id ภายใน | ใส่อะไร |
|---------|----------|---------|
| มือหลัก | `main_hand` | อาวุธมือเดียว / สองมือ / บางครั้ง thruster |
| มือรอง | `off_hand` | อาวุธมือเดียว / โล่ / **ว่าง** (ถ้ส main เป็นสองมือ → ล็อก) |
| ศีรษะ | `head` | หมวก · หน้ากาก · มงกุฎ |
| ลำตัว | `body` | เกราะอก · ชุดคลุม |
| ส่วนล่าง | `legs` | กางเกง · กระโปรงเกราะ |
| เท้า | `feet` | รองเท้า · เกราะเท้า |
| เครื่องประดับ | `acc_1` … `acc_N` | แหวน สร้อย (เริ่ม N=1 หรือ 2) |

**Migrate จากของเก่า:**

| เดิม | ใหม่ |
|------|------|
| `weapon` | → `main_hand` |
| `armor` | → `body` (ของเก่ายังใส่ body ได้) |
| `accessory` | → `acc_1` |

### 3.2 โหมดอาวุธ (grip / stance)

ทุกอาวุธมี `grip` (หรือ derive จาก tags):

| grip | ความหมาย | กฎช่อง |
|------|----------|--------|
| `one_hand` | มือเดียว | ใส่ main **หรือ** off · **dual:** main+off เป็น one_hand ทั้งคู่ |
| `two_hand` | สองมือ | ใส่ main เท่านั้น · **บังคับว่าง off_hand** |
| `shield` | โล่ | ใส่ **off_hand เท่านั้น** |
| `focus` (optional) | คทา/หนังสือเวทมือรอง | off_hand · soft เวท |

**ชุดที่ถูกต้อง (valid loadouts):**

| แบบ | main | off | soft ผลโดยประมาณ |
|-----|------|-----|-------------------|
| มือเดียว + โล่ | one_hand | shield | กันกาย soft ดี · ดาเมจปานกลาง |
| คู่ดาบ (dual) | one_hand | one_hand | โจมตี/ATB soft ดี · กันอ่อน |
| สองมือ | two_hand | *(ล็อก)* | ดาเมจสูง · ช้า/ช่องว่างกัน |
| มือเดียวอย่างเดียว | one_hand | ว่าง | สมดุลกลาง · มือรองว่าง (บัฟภายหลัง?) |
| เวท + focus | one_hand/catha | focus | soft เวท · กันกายอ่อน |

**ตอนสวม conflicting:**
- ใส่ two_hand ขณะมีของ off → **ถอด off กลับกระเป๋า** + soft 「ต้องใช้สองมือ」
- ใส่โล่ขณะ main เป็น two_hand → ปฏิเสธหรือถอด two_hand

### 3.3 Schema ไอเทม (YAML)

```yaml
- id: iron_sword
  kind: equipment
  slot: main_hand          # หรือ slot_group: weapon
  grip: one_hand
  # อนุญาต dual
  allow_off_hand: true
  sockets: 1
  # soft biases (ไม่โชว์ตัวเลข)
  bias:
    damage_class: physical
    guard_physical: slight   # slight|mid|strong — map เป็นค่าซ่อน
    atb: neutral

- id: oak_greatsword
  slot: main_hand
  grip: two_hand
  allow_off_hand: false
  bias:
    power: high
    atb: slow

- id: wood_shield
  slot: off_hand
  grip: shield
  bias:
    guard_physical: strong
    guard_arcane: slight

- id: iron_helm
  slot: head
  bias:
    guard_physical: slight
    status_resist: slight

- id: leather_chest
  slot: body
  # migrate: เดิม slot armor

- id: greaves
  slot: legs

- id: trail_boots
  slot: feet
  bias:
    atb: slight_fast
```

### 3.4 ซ็อกเก็ตการ์ด

- คงโมเดล “การ์ดใส่ในช่องบนชิ้นเกียร์”
- ช่องใหม่ (head/body/…) มี `sockets: 0–2` ตาม rarity soft
- two_hand อาจมี sockets มากกว่า one_hand (trade-off)

### 3.5 ชุด (set) ข้ามช่อง

เซ็ตเดิม (shadow / thorn) ขยาย:  
นับชิ้นจาก **หลายช่อง** (หัว+ตัว+ขา+เท้า+มือ)  
ยัง soft flavor — ไม่โชว์ “2/4 pieces +8%”

| ช่อง / grip | โน้ม soft ไปทาง |
|-------------|-----------------|
| โล่ off | def กาย · กันสกิลกาย |

| หมวก/ชุด | def กาย + ต้านสถานะ soft |

## 5. UI soft (เกียร์)

### 5.1 แผงสวม (เป้าหมาย)

```text
┌ เกียร์ ───────────────────────────────────┐
│ ศีรษะ      …                               │
│ มือหลัก    …     มือรอง    … / โล่ / ล็อก  │
│ ลำตัว      …                               │
│ ส่วนล่าง    …                               │
│ เท้า       …                               │
│ เครื่องประดับ …                            │
├ soft ─────────────────────────────────────┤
│ 「ถือสองมือ — มือรองว่าง」                  │
│ 「โล่ทำให้ก้าวช้าลงเล็กน้อย」                │
└────────────────────────────────────────────┘
```

เลือกช่อง: **เลข** (1 ศีรษะ · 2 มือหลัก · 3 มือรอง · …)  
ไม่บังคับพิมพ์ `weapon/armor`

### 5.2 ตอน conflict

```text
┌ ใส่ไม่ได้แบบนี้ ──────────────────────────┐
│ ดาบใหญ่ต้องการสองมือ                       │
│  1  ถอดของมือรองแล้วใส่                    │
│  0  ยกเลิก                                 │
└────────────────────────────────────────────┘
```

## 6. แผนเฟส implement (Equip Loadout)

| เฟส | ชื่อ | งาน | เป้า |
|-----|------|-----|------|
| **EL0** | Schema + migrate | ช่องใหม่ใน save · map weapon→main_hand, armor→body | เซฟเก่ายังเล่นได้ |
| **EL1** | Grip rules | one_hand / two_hand / shield · dual · ล็อก off | กฎสวมครบ |
| **EL2** | Content slots | แยก body/head/legs/feet ไอเทมใหม่ + migrate armor | แต่งตัว 4 ส่วน |
| **EL3** | Soft bias recompute | bias → power_def/mdef/atb soft (คู่ DD1) | ชุดต่างรู้สึกต่าง |
| **EL4** | UI เกียร์เต็ม | แผงหลายช่อง · เลขเลือก · conflict dialog | UX ครบ |
| **EL5** | Set + market | เซ็ตข้ามช่อง · ตลาดรองรับ slot ใหม่ | เศรษฐกิจ |

**ลำดับแนะนำ:**  
EL0 → EL1 → EL4 (UI บาง) → EL2 → EL3 (คู่ DD1) → EL5  

**พึ่งพา DD:**  
- EL3 ควรหลังหรือขนาน **DD1** (mdef)  
- ไม่ต้องรอ DD3–DD5 ครบถึงเริ่ม EL0–EL1

```text
DD0–DD1 (ชั้นดาเมจ/กัน)
    \\
     EL0 → EL1 → EL2 → EL4
              \\
               EL3 (bias) → EL5
```

## 12. วิเคราะห์ช่องลึก (ตามสเปกผู้เล่น)

### 12.1 อาวุธ — มือเดียว · สองข้าง · สองมือ · มือเดียว+โล่

```text
                    main_hand
                   /    |    \
           one_hand  two_hand  (อื่น)
              |         |
     ┌────────┼────┐    off ล็อก
     │        │    │
  off ว่าง  off 1H  off shield
  (สมดุล)   (dual)  (1H+โล่)
```

| โหมด | main | off | ข้อได้ soft | ข้อเสีย soft | ใครมัก “รู้สึก” เข้า |
|------|------|-----|-------------|--------------|---------------------|
| **1H อย่างเดียว** | one_hand | ว่าง | กลาง · ยืดหยุ่นสลับ | ไม่สุดทั้งกัน/ดาเมจ | มือใหม่ · เปลี่ยนชุดบ่อย |
| **1H + โล่** | one_hand | shield | กันกาย/กันสกิล · ทน | ATB ช้า · ดาเมจแผ่ว · น้ำหนัก | แนว tank / ดันยาว |
| **Dual (1H+1H)** | one_hand | one_hand | โจมตี/จังหวะ · combo soft | กันอ่อน · off ดาเมจแผ่ว · เหนื่อย needs | แนว aggression / speed |
| **Two-hand** | two_hand | *ล็อก* | power สูง · ซ็อกเก็ตเยอะ | ไม่มีโล่ · ช้า · ใส่เกราะหนักแล้วหนักซ้อน | แนว power / boss burst |
| **1H + focus** (หลัง) | one_hand | focus | เวท/ม่าน soft | กันกายอ่อน | นักเวท · light/dark |

**กฎสมดุลซ่อน (อย่าโชว์ตัวเลข):**

| กฎ | ผลซ่อนโดยประมาณ |
|----|------------------|
| Off-hand อาวุธ | ดาเมจ off ≈ **45–65%** ของ main (noise ตาม instance) |
| Dual ATB | ช้ากว่า 1H+โล่เล็กน้อย **หรือ** เร็วแต่ stamina/needs กินเร็ว — เลือกทิศหนึ่งตอน EL3 |
| โล่ | +guard_physical soft · ลด ATB · บางโล่ +elem_resist แผ่ว |
| Two-hand | +power · บังคับล็อก off · อาจ +reach/AoE soft บนสกิลบางอัน |
| มือว่าง off | โอกาส soft “มือว่าง = เปลี่ยนของเร็ว” (EL5+) ไม่บังคับ |

**สิ่งที่เกมอื่นทำให้ “รู้ทันที” แต่เราไม่ทำ:**
- ไม่โชว์ DPS เปรียบ dual vs 2H
- ไม่มีตาราง “โล่ +12% ลดดาเมจ”
- dual ไม่ได้ = 2× ดาเมจเสมอ (off แผ่ว + soft cap)

### 12.2 ชุดสวมใส่ — หัว · ลำตัว · ส่วนล่าง · รองเท้า

แยกเกราะเดิม `armor` เดียว → **4 ส่วน** เพื่อให้ “ชุดเดียวกัน” ประกอบได้หลายทาง  
**แต่ละส่วนมีบทบาทหลัก soft** แต่ **ห้าม 1:1 ตายตัว** (S-P2)

| ช่อง | id | บทบาทหลัก soft | บทบาทรอง (สุ่ม/แท็ก) | ตัวอย่าง bias |
|------|-----|----------------|----------------------|---------------|
| **ศีรษะ** | `head` | ต้านสถานะ · ม่านสติ/เวทแผ่ว | crit รับ soft · perception | helm หนัก = กันกาย · hood = mdef/curiosity soft |
| **ลำตัว** | `body` | กันหลัก (กาย/เวท ตามวัสดุ) | น้ำหนักทั้งตัว | plate = def · robe = mdef · leather = กลาง+ATB |
| **ส่วนล่าง** | `legs` | กันรอง + เคลื่อนที่ | ต้าน knock/slow | greaves หนัก = กัน · ผ้า = เร็ว |
| **รองเท้า** | `feet` | ATB · หลบ · เดินทางสนาม | ต้านพื้นที่ (โคลน/น้ำแข็ง soft) | boots เบา = เร็ว · เกราะเท้า = กันเล็ก |

**สัดส่วนกันซ่อน (แนะนำเริ่ม — ปรับได้ใน YAML balance):**

```text
body  ≈ 40–45% ของเกราะรวม
head  ≈ 15–20%
legs  ≈ 20–25%
feet  ≈ 10–15%
(+ โล่ / ชุดเซ็ต / บัฟ อยู่นอกสัดส่วนนี้)
```

ผู้เล่น**ไม่เห็น %** — รู้แค่ soft 「ชุดอกหนัก รับกระแทกได้」「รองเท้าเบา ก้าวไว」

**ทำไมแยก 4 ส่วนถึงคาดเดายาก:**
1. ใส่ plate อก + leather ขา + รองเท้าเบา → hybrid ที่ wiki จัด meta ยาก  
2. เซ็ตไม่ครบยังมีผล soft แผ่ว (ไม่ใช่ all-or-nothing)  
3. instance rarity ต่อชิ้น → “หมวกชื่อเดียวกัน” ไม่เท่ากัน  
4. วัสดุปะทะพื้นที่ (S-P12) → ชุดเดียวกัน ป่า ≠ ดันไฟ  

### 12.3 เครื่องประดับ (acc)

| ชั้น | นโยบาย |
|------|--------|
| EL0–EL2 | `acc_1` ชิ้นเดียว (migrate จาก accessory) |
| EL5+ | `acc_2` เปิดเมื่อ mastery/อาชีพ/พร soft — ไม่โชว์เงื่อนไขครบ |
| บทบาท | เวท · แสง/มืด · luck · ต้านสถานะเฉพาะ — **ไม่** แทนเกราะหลัก |

## 13. ระบบเพิ่มเติมที่ทำให้ “ไม่รู้ทันทีว่าต้องแต่งยังไง”

หลัก: **เกียร์อย่างเดียวไม่พอ** — หลายระบบทับซ้อนแบบ soft  
ถ้ามีแค่ ATK/DEF บนเกียร์ → กลับไป meta เดิม

### 13.1 แผนที่ระบบ (Multi-system stack)

```text
┌──────────────────────────────────────────────────────────┐
│  ผลไฟต์ / สำรวจ ที่ผู้เล่น “รู้สึก”                          │
│  = f( เกียร์×grip , ชั้นดาเมจ , อาชีพ latent , นิสัย ,      │
│       needs , พื้นที่ , instance , โชค , ปาร์ตี้ , เวลา )   │
└──────────────────────────────────────────────────────────┘
         ▲ ทุกชั้น soft / diminishing / context
```

### 13.2 ระบบเสริม — แคตตาล็อกออกแบบ

| # | ระบบ | id แผน | ทำไมคาดเดายาก | เฟสผูก |
|---|------|--------|---------------|--------|
| **A** | **น้ำหนักชุด (weight class)** | `EQ-W` | เบา/กลาง/หนัก soft — เร็ว vs ทน ไม่โชว์ kg | EL3 |
| **B** | **ความเข้ากับ grip (stance affinity)** | `EQ-G` | ใช้ dual บ่อย → soft ชิน dual · สลับ 2H แล้ว “ยังไม่ชิน” ชั่วคราว | EL3–EL5 |
| **C** | **อาชีพ × ช่อง (occ soft)** | `EQ-O` | นักรบ โล่/2H soft ดี · นักเวท robe+focus — **ไม่** ล็อก equip | ขนาน occ |
| **D** | **นิสัย × สไตล์สวม** | `EQ-P` | courage โน้ม dual/2H · caution โน้มโล่ — soft เล็ก | personality |
| **E** | **Needs × น้ำหนัก** | `EQ-N` | ชุดหนัก + หิว/เหนื่อย → ATB/ดาเมจแผ่วชัด | Tama + EL3 |
| **F** | **พื้นที่ × วัสดุ** | `EQ-A` | เกราะโลหะในหนอง/พายุ soft แย่ · ผ้าในความเย็น soft แย่ | world tags |
| **G** | **เซ็ตไม่สมบูรณ์ + ปะปน** | `EQ-S` | 2/4 soft แผ่ว · ปะวัสดุต่าง = noise ไม่ใช่ bonus ตายตัว | EL5 |
| **H** | **การ์ด/ซ็อกเก็ตตำแหน่ง** | `EQ-C` | การ์ดใบเดียวกันบน head ≠ บน weapon (bias ต่าง) | sockets |
| **I** | **สถานการณ์ล็อกเกียร์** | `EQ-L` | ในไฟต์/ดันลึก เปลี่ยนเกียร์ช้าหรือไม่ได้ — ต้องเดาก่อนเข้า | combat/dungeon |
| **J** | **คราฟเสี่ยง + instance** | `EQ-I` | คราฟได้ “ใกล้เคียง” ไม่ใช่ copy meta · quality soft | craft K |
| **K** | **ตลาดภาษีของดี** | `EQ-M` | ของแรงอยู่ตลาด ไม่ใช่ร้าน — หาไม่ชัวร์ | market |
| **L** | **พร / latent seed** | `EQ-B` | ตัวละครคนละ seed · ชุดเดียวกันผลแผ่วต่าง | blessing |

### 13.3 น้ำหนักชุด (A) — รายละเอียด

| weight_class soft | มาจาก | ผล soft โดยประมาณ |
|-------------------|--------|-------------------|
| เบา `light` | leather/cloth หลายช่อง · dual เบา | ATB ดี · กันอ่อน · needs ช้า |
| กลาง `medium` | ผสม | สมดุล |
| หนัก `heavy` | plate + โล่ / 2H เหล็ก | กันดี · ช้า · needs กินเร็ว · บางพื้นที่ย่ำแย่ |

UI: 「ก้าวหนัก」「ตัวเบา」— ไม่โชว์ “weight: 47”

### 13.3b ค่าหลักที่โชว์ · พลังแฝงที่ต้องสังเกต

| ชนิด | UI โชว์ | ซ่อน (latent · rank scale) |
|------|---------|---------------------------|
| **อาวุธ** | โจมตี (ATK) · MP ถ้ามี | `latent_atk_pct` · `latent_crit` — แรงขึ้นตาม rarity |
| **เกราะ/โล่** | กันกาย · กันเวท | `latent_hp_pct` · `latent_tough` · `latent_status_resist` (อึด/ต้าน) |

- ชิ้นละค่า latent ไม่เท่ากัน (YAML + default ตาม weight/slot + `_latent_rank_scale`)
- ผู้เล่น**ไม่**เห็น % แฝง — สังเกตเลือดรวม · แรงโจมตี · คริ · ต้านสถานะในไฟต์
- soft examine: 「…คมแฝง…」「…เกราะอุ้มร่าง/อึด…」

### 13.3c พิธีอัปเกรดเสี่ยง · Rank

| ผล | ความหมาย | กันด้วย |
|----|----------|---------|
| สำเร็จ | +level · primary + latent แน่นขึ้น | — |
| ล้ม soft | ขั้นเดิม · เสีย mats/เงิน | — |
| ล้มร้ายแรง | ระดับ + ลด 1 | `scroll_guard_level` **rank ≥ ชิ้น** |
| ล้มร้ายแรงสุด | ชิ้นพัง (การ์ดคืนถุง) | `scroll_guard_break` **rank ≥ ชิ้น** |

**ปัจจัยอัตรา (ซ่อน %):**

| แกน | ผล |
|-----|-----|
| **+level** | โอกาสสำเร็จลด · fail รุนแรงขึ้น · ราคาแพง |
| **Rank ชิ้น** | โอกาสสำเร็จลดเพิ่ม · break/down หนัก · เงิน/mats แพง |
| **upgrade_bonus** (tier YAML) | ชดเชยแผ่วจากคุณภาพ — ไม่ชนะ rank penalty |
| **ม้วน** | ใช้ได้เมื่อ **rank ม้วน ≥ rank ชิ้น** · ม้วนสูงกว่าใช้กับชิ้นต่ำได้ (กินม้วนดี) |

- ม้วนในถุงใช้**อัตโนมัติ**เมื่อ severity ตรง (ไม่บังคับมีก่อนอัป)
- ร้านวัสดุขายม้วนหลาย rank (common → sacred)

### 13.4 Grip affinity (B) — รายละเอียด

```text
stance_meters (ซ่อน):
  one_hand_shield: 0..100
  dual: 0..100
  two_hand: 0..100
```

- ชนะไฟต์ในโหมดนั้น → meter โหมดนั้นขึ้นช้า  
- สลับโหมดที่ meter ต่ำ → soft 「มือยังจำท่าเดิม」(ดาเมจ/กันแผ่ว 1–2 ไฟต์ หรือจน meter ตามทัน)  
- **ห้าม** โชว์ meter — ใบ้แค่ flavor หลังไฟต์บางครั้ง  

→ ผู้เล่น**ไม่**เปลี่ยน build ทุก 10 วินาทีตาม wiki โดยไร้ค่า

### 13.5 อาชีพ / นิสัย (C·D) — ไม่ล็อกเกียร์

| ผิด (เกมทั่วไป) | ถูก (เกมนี้) |
|-----------------|-------------|
| “นักเวทใส่เกราะไม่ได้” | ใส่ได้ แต่ soft เวทแผ่ว / ATB หนัก |
| “class armor only” | ทุกอาชีพใส่ทุกช่องได้ |
| “recommended gear” ใน UI | ไม่มี — มีแค่ soft หลังทดลอง |

ตัวอย่าง soft occ bias (YAML ซ่อน):

```yaml
warrior:
  soft_affinity:
    grip: [shield, two_hand]
    slots: { body: plate_bias }
mage:
  soft_affinity:
    grip: [focus, one_hand]
    slots: { body: cloth_bias, acc: arcane }
```

### 13.6 Needs × เกียร์ (E)

| สภาพ | ชุดเบา | ชุดหนัก |
|------|--------|---------|
| อิ่ม/สด | ปกติ | ปกติ (ช้ากว่าเล็ก) |
| หิว/เหนื่อย | ยังไหว soft | 「เกราะทับไหล่」ชัด — ดาเมจรับเพิ่ม / ATB ช้า |
| หนาว/ร้อน (พื้นที่) | ตามวัสดุ | ตามวัสดุ + น้ำหนัก |

ผูก `NEEDS_COMBAT_FOOD_VISION` / Tama — เกียร์ไม่แยกจากชีวิตตัวละคร

### 13.7 พื้นที่ × วัสดุ (F)

แท็กไอเทม: `material: metal|leather|cloth|bone|crystal|wood`  
แท็กพื้นที่: `climate: wet|arid|cold|hot|dark|holy_ground|…`

```text
match soft:
  metal + wet     → กัน/ATB แผ่วเล็ก 「สนิมในใจ」
  cloth + cold    → HP soft / status chill ง่าย
  crystal + dark  → เวท soft ดีหรือแย่ตามธาตุ
```

**ไม่มี** ตารางครบในเกม — ห้องสมุด fragment ใบ้บางส่วน (เหมือน personality)

### 13.8 ทำไมทั้งก้อนถึง “ต่างจากเกมที่รู้ทันที”

| เกมที่รู้ทันที | เกมนี้ |
|----------------|--------|
| เปิดแผง +12 ATK เลือกสูงสุด | soft bias หลายแกน + diminishing |
| 1 meta dual wield | dual แผ่ว off + affinity + needs |
| ชุดเซ็ต 4/4 บังคับ | ปะปนได้ · เซ็ตไม่ครบยังมีค่า |
| class ล็อกเกียร์ | อาชีพแค่โน้ม soft |
| wiki element chart | matchup soft · ไม่โชว์ % |
| respec ฟรีในเมืองตลอด | เปลี่ยน grip มี “ความชิน” ชั่วคราว |
| shop ขายของ BiS | ร้าน common · ของดี drop/ตลาด |

**คำตอบของเกมเมื่อถาม “ใส่ชุดไหนดี?”**  
→ *ขึ้นกับป่ามืดหรือดันไฟ · หิวไหม · มือชินโล่หรือยัง · วันนี้โชค/พร · ลองแล้วดู soft*
