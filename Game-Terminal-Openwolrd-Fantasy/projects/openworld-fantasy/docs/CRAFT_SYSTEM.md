## 2. กฎที่ implement แล้ว (เงื่อนไข rank)

```text
ถ้า inputs_rarity[item] = uncommon
  → ใช้ได้เฉพาะชิ้นที่ rarity rank ≥ 「สูง」
  → ชิ้น common ใช้ไม่ได้ (ข้อความ soft: วัตถุดิบระดับไม่พอ)
  → หักจากชิ้นระดับต่ำสุดที่ยังพอ (≥ ที่กำหนด)
```

| ชั้น | ความหมาย |
|------|----------|
| `unlock_level` | เลเวลผู้เล่นขั้นต่ำ |
| `inputs` | จำนวนวัตถุดิบ |
| `inputs_rarity` | ขั้นต่ำเกรดต่อ id |
| `money` | ค่าธรรมเนียมคราฟ |
| `output` + `output_rarity` | ผลและเกรดบังคับ |

**ตอนนี้ (1.14.1):** เงื่อนไขผ่าน → **roll สำเร็จ** (base ตาม output tier · โบนัส mat เกรดสูง · soft label)

## 4. ออกแบบ: โอกาสสร้างสำเร็จ (เสนอ — ยังไม่โค้ด)

### 4.1 หลัก

| รหัส | หลัก |
|------|------|
| C1 | Soft / anti-spoiler — **ไม่โชว์ % ดิบ** ใช้ป้าย “นิ่ง / สั่น / เสี่ยง” |
| C2 | วัตถุดิบ **ต่ำกว่า min** → ยังคราฟไม่ได้ (กฎเดิม) |
| C3 | วัตถุดิบ **สูงกว่า min** → โอกาสดีขึ้น (ไม่การันตี) |
| C4 | ผล output แรงก์สูง → base chance ต่ำลง |
| C5 | ล้มเหลวไม่ softlock — mat บางส่วนคืน / เงินคืนบางส่วน |
| C6 | แยกจาก **อัปเกียร์สวม** (คนละ curve) |

### 4.2 สูตรโอกาส (ซ่อน)

```text
P_success = clamp(
    base_by_output_tier
  * material_quality_mult   # เฉลี่ย (actual_rank - min_rank)
  * station_mult            # optional
  * level_soft              # เลเวลเกิน unlock เล็กน้อย
  * noise                   # hash รายวัน 0.92–1.08
, 0.12, 0.97)
```

| `output_rarity` | base (แนวทาง) |
|-----------------|---------------:|
| common | 0.92 |
| uncommon | 0.82 |
| rare | 0.68 |
| sacred | 0.52 |
| legendary+ | 0.38 |

| วัตถุดิบเทียบ min | material_quality_mult |
|-------------------|----------------------:|
| เท่า min พอดี | 1.00 |
| +1 tier เฉลี่ย | 1.08 |
| +2 tier | 1.15 (cap) |
| หลายชิ้นแค่ min | 0.95 soft |

### 4.3 ผลเมื่อล้มเหลว (soft)

| แบบ | โอกาสภายใน | ผู้เล่นเห็น |
|-----|------------|-------------|
| soft_fail | ส่วนใหญ่ | เสีย mat ครึ่ง / เงินครึ่ง · 「โลหะไม่ยอมจับรูป」 |
| hard_fail | หาง | เสีย mat ทั้งหมด · 「เถ้าเท่านั้นที่เหลือ」 |
| partial_product | หายาก | ได้ mat เศษ / echo_shard แทน |

**อย่า** ลบเงินทั้งหมด + mat ทั้งหมดทุกครั้ง — โหดเกิน alpha

### 4.4 YAML ขยาย (เสนอ)

```yaml
- id: craft_shadow_dagger
  # ... เดิม ...
  success:
    base: 0.68          # optional override
    fail_mode: soft     # soft | hard | soft_then_hard
    refund_money_pct: 0.4
    refund_mat_pct: 0.5
  # หรือใช้ defaults จาก output_rarity ใน craft_rules.yaml
```

ไฟล์ใหม่ optional: `data/craft/rules.yaml` (base chance ต่อ tier · fail policy)

### 4.5 UI soft

```text
  5. หลอมกริชเงา | … → กริชเงา [◆]
     เงื่อนไข: ดาบเหล็ก≥สูง · …
     รู้สึก: 「โอกาสปานกลาง」 / 「โลหะยังไม่นิ่ง」
```

แมป label จาก P: ≥0.8 สูง · ≥0.55 ปานกลาง · <0.55 เสี่ยง — เหมือนอัปเกียร์

### 4.6 สิ่งที่ไม่คราฟ (นโยบายเศรษฐกิจ)

| กลุ่ม | เหตุผล |
|-------|--------|
| unit unique | ชิ้นเดียวโลก/เซฟ — ดรอปหีบพันธะเท่านั้น |
| sealed chest | ได้จากบอส/เควส ไม่คราฟ |
| void_key | เนื้อหาซ่อน |

คราฟ sacred (thorn) **โอเค** — เป็นทาง “ลงทุน mat + rank อาวุธ” แทนร้านระบบ

## 6. ทางเล่นในเกมตอนนี้

```text
สำรวจ / เมือง → เมนูคราฟ (C จากกระเป๋าหรือ field)
  → เห็นสถานีของพื้นที่นี้ · สูตรที่สถานีนี้รองรับ
  → เลือกสูตรที่ unlock ตามเลเวล
  → ต้องมีวัตถุดิบ + เงิน + เกรดอาวุธ/mat พอ
  → เห็น soft โอกาส (สูง/ปานกลาง/เสี่ยง) → roll สำเร็จหรือล้ม (soft คืนบางส่วน)
  → สูตรสถานีอื่น: soft ใบ้ให้เดินทาง
```

เควส: `forge_initiate` (คราฟ 1 ครั้ง) · `forge_bread` (คราฟ 2 ครั้ง)
