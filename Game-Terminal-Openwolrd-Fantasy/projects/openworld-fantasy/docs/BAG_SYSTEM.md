# ระบบกระเป๋าแบบแยกประเภท (Bag Categories)

**สถานะ:** Design + implement v1.2 (1.10.4 · equip short codes)  
**ปัญหาที่แก้:** เมนู 5 เดิมรวมทุกอย่าง · อุปกรณ์ไม่โชว์ไอดี/ระดับ · ของสวมไม่พิมพ์ดูได้  

---

## 1. ปัญหาปัจจุบัน

```text
สนาม → 5
  คลัง: ยา, ดาบ, วัสดุ ปนในบรรทัดเดียว
  1.ใช้ยา  2.อุปกรณ์/การ์ด/ร้าน
```

| ปัญหา | ผล |
|--------|-----|
| ไม่มี hub “ประเภท” | ผู้เล่นไม่รู้ว่าใช้ยาอยู่ไหน |
| รายการแบนราบ | ยาปนวัสดุปนเกียร์ |
| ร้าน/คราฟซ่อนในเกียร์ | ลึกเกิน |
| co-op ต้องจำ 51 | ไม่ใช่คำสั่งเกมจริง |

---

## 2. หลักการออกแบบ

1. **เลือกประเภทก่อน แล้วค่อยเลือกชิ้น** (2 ขั้นชัด)  
2. **ประเภท = วิธีใช้** ไม่ใช่แค่ tag สวยๆ  
3. **แสดงจำนวนต่อประเภท** บน hub  
4. **การ์ดแยกถุง** (card_bag) อยู่หมวดตัวเอง  
5. **ของที่สวมอยู่** ดูได้จาก hub โดยไม่ต้องไล่คลัง  
6. soft: ชื่อ + rarity · สูตรคราฟยังไม่สปอยล์  

---

## 3. หมวดประเภท (Categories)

| รหัส | ชื่อไทย | เกณฑ์จัด | แอคชันหลัก |
|------|---------|----------|------------|
| **equipment** | อุปกรณ์ | `kind=equipment` หรือมี `slot` | สวม / ดู |
| **food** | อาหาร | `tags: food` / food_tier / hunger_relief | **กิน** → อิ่มท้อง |
| **healing** | รักษา | ยา heal / clear / apply_status (ไม่ใช่อาหาร) | ใช้ทันที |
| **material** | วัตถุดิบ | `kind=material` | ดู (คราฟ/อัป) |
| **card** | การ์ด | `card_bag` | ใส่ช่องเกียร์ |
| **other** | อื่นๆ | ที่เหลือ | ดู / ใช้ |
| **equipped** | ที่สวมอยู่ | equip_ids | แผงเกียร์ |
| **services** | บริการ | — | ร้าน · คราฟ · ทั้งหมด |

### กฎจัดหมวด (ลำดับตัดสิน)

```text
1. card_bag          → card
2. kind == equipment หรือ slot → equipment
3. kind == material  → material
4. is_food_item (tags food / food_tier / เสบียง) → food
5. heal / clear / apply_status → healing
6. kind == consumable → healing (fallback)
7. else              → other
```

### เมนู hub (1.13.16+)

```text
 1 อุปกรณ์  2 อาหาร  3 รักษา  4 วัตถุดิบ  5 การ์ด  6 อื่นๆ
 7 ที่สวม  8 ร้าน  9 คราฟ  A ทั้งหมด
```

### 4.1 Hub กระเป๋า (กด 5 จากสนาม)

```text
── กระเป๋า (5/40) ──
 สวมอยู่ (พิมพ์ไอดีเพื่อดู/จัดการ):
   อาวุธ: sw001 ดาบเหล็ก [○] ธรรมดา
   เกราะ: —
   เครื่องประดับ: —
 ระดับ: [○]=ธรรมดา [◇]=สูง [◆]=หายาก …   ← อ่านชื่อไทยคู่สัญลักษณ์
  ตัวอย่าง: พิมพ์ sw001 → ดูดาบที่สวม

 1. อุปกรณ์     (0)
 2. รักษา       (3)
 ...
 0. กลับสนาม

เลือกประเภทหรือไอดี (เช่น sw001): _
```

```text
 4. ขาย
 5. ทิ้ง
```

| ไอดีสั้น | ชิ้น |
|----------|------|
| sw001… | อาวุธ (ดาบ/ไม้เท้า/…) |
| ar001… | เกราะ |
| ac001… | เครื่องประดับ |

`code` ใน `data/items/items.yaml` · โค้ด `game/domain/item_codes.py`

### 4.3 หมวดอุปกรณ์

```text
── อุปกรณ์ ──
  (แสดง ไอดี · ชื่อ · ระดับอาวุธ — พิมพ์หมายเลขหรือไอดี)
  1. iron_sword  ดาบเหล็ก [○ธรรมดา]  [Lv.1 ○ธรรมดา]  ATK+6 · อาวุธ
  2. steel_blade ดาบเหล็กกล้า · สูง  [Lv.2 ◇สูง]  ATK+11 · อาวุธ
  0. กลับ

เลือกชิ้น (หมายเลขหรือไอดี · 0=กลับ): iron_sword
→ แผง showcase ตามระดับอาวุธ (กรอบ/อรรถรส/ASCII ยิ่งสูงยิ่งอลัง)
```

**ระดับอาวุธ (rarity rank)** → ความอลังของ text UI:

| Lv | id | ชื่อ | กรอบ text |
|----|-----|------|-----------|
| 1 | common | ธรรมดา | `┌─┐` เรียบ + ดาบมินิ |
| 2 | uncommon | สูง | มุม `◇` |
| 3 | rare | หายาก | `◆` + แถบ · |
| 4 | sacred | ศักดิ์สิทธิ์ | `★` + ใบมีดประดับ |
| 5 | legendary | ตำนาน | แบนเนอร์ `✦` + ⚔ |
| 6 | divine | เทพ | แถบ ░▒▓ |
| 7 | archdivine | มหาเทพ | `✪` + ออร่า |
| 8 | mythic | ปฐมกาล | `◈` สูงสุด |

### 4.4 หมวดวัตถุดิบ

```text

  1. วัสดุอัพเกรด x2  (ใช้ตอนอัปเกียร์/คราฟ)
  0. กลับ
  (ใช้ตรงจากที่นี่ไม่ได้ — ไปคราฟหรืออัปในเกียร์)
```

### 4.4b ยืนยัน y/n ก่อนแอคชันเสี่ยง (1.10.4+)

หลังเลือกแล้ว **ระบบสรุปแล้วถาม** ก่อน commit:

```text
 จะใส่: การ์ดไฟ → อาวุธ ช่อง 1
ยืนยันใส่การ์ด? (y=ตกลง / n=ยกเลิก): _
```

| แอคชัน | ยืนยัน |
|--------|--------|
| ใส่การ์ด | y/n |
| สวมอุปกรณ์ | y/n |
| ถอด / อัป / ขาย / ทิ้ง (sw001) | y/n |

`y` · `ใช่` · `1` = ตกลง · อื่นๆ รวม `n` = ยกเลิก

### 4.4c อัปเกรดอุปกรณ์ (preview + เสี่ยง + ม้วนกัน)

- **อัปได้** (สวมอยู่ + ระดับ &lt; ขีด) → แสดง `3. อัพเกรด`
- **อัปไม่ได้** (ถึงขีด / ว่าง) → **ไม่แสดง** เมนูอัป
- กด `3` → แผงค่าใช้จ่าย + ความเสี่ยง + ม้วนคุ้มครอง แล้ว y/n
- **ผลพิธี:** สำเร็จ · ล้ม (เสียวัสดุ) · ล้มร้ายแรง (ลด +level) · ล้มร้ายแรงสุด (ชิ้นพัง)
- **ม้วน:** `scroll_guard_level` กันลดระดับ · `scroll_guard_break` กันพัง
  - **Rank ม้วนต้อง ≥ Rank ชิ้น** (ม้วนตำนานใช้กับชิ้นธรรมดาได้ · กลับกันไม่ได้)
  - ใช้เมื่อล้มร้ายแรง**อัตโนมัติ** (ไม่บังคับมีก่อนอัป)
- ค่าใช้จ่ายขึ้นตาม **+level + Rank ชิ้น** · โอกาส/ความเสี่ยง soft label เท่านั้น

```text
── พิธีอัปเกรด ──
 ชิ้น: sw001 ดาบเหล็ก  (มือหลัก)
 ระดับอัป: +0  →  เป้า +1
 ค่าใช้จ่าย (หักทันทีเมื่อยืนยัน — แม้พิธีล้มเหลวก็เสีย):
  · เงินโลก: 40  (มี 82)  ✓ พอ
  · วัสดุอัพเกรด [… / upgrade_mat]: ×1  (มี 0)  ✗ ไม่พอ
 โอกาสสำเร็จ (ประมาณ): โอกาสสูง
 ความเสี่ยงเมื่อล้ม: ล้ม = เสียวัสดุ (ลดระดับ/พังยังแทบไม่มี)
 ม้วนคุ้มครอง: ม้วนกันลดระดับ · ม้วนกันพัง (ถ้ามีในถุง)
ยืนยันเริ่มพิธีอัปเกรด? (y=ตกลง / n=ยกเลิก): _
```

## 5. การไหล (Flow)

```text
สนาม [5]
   │
   ▼
Hub ประเภท ──1──► รายการอุปกรณ์ ──► สวม/ดู
   ├────2──► รายการรักษา ──► ใช้ 1 ชิ้น ──► กลับ hub หรือสนาม
   ├────3──► วัตถุดิบ (ดู)
   ├────4──► การ์ด ──► ใส่ช่อง
   ├────6──► แผงสวมใส่ + อัป/ถอดการ์ด
   ├────7──► ร้าน
   └────8──► คราฟ
```

**ในไฟต์:** คงเมนู 3 ยา/ล้าง/บัฟ (รายการเฉพาะ healing) — ไม่ต้องเปิด hub เต็ม

## 7. ความสัมพันธ์ข้อมูล

```text
inventory_ids[]  + inventory[]  + inventory_rarities[]  + inventory_qty[]  ← ของทั่วไป (1 แถว = 1 ช่อง)
inventory_items[].qty                                                   ← instance SoT (WO-INV-1)
card_bag[]       + card_rarities[]                                      ← การ์ด (1 ใบ = 1 ช่อง)
equip_ids / sockets / upgrade_levels                                    ← สวมอยู่
```

**WO-INV True stack:** consumable / material / food / healing / sealed chest  
→ ชนิดเดียวกัน + rarity เดียวกัน = ช่องเดียว มี `qty`  
→ อุปกรณ์ / เรลิก / quest ไม่ stack  

**Soft cap:** นับ *ช่อง* (`bag_count` = stacks + cards) · เกิน cap แล้วเพิ่มช่องใหม่ไม่ได้  
(stack เข้ากองเดิมได้แม้เต็ม) · `add_item` / `try_add_item` / ร้านซื้อ เคารพ cap  

**Organize (hub `O`):** `bag_organize.organize_bag` — collapse stack แล้วเรียง หมวด → rarity → ชื่อ  

**Bulk sell (ร้าน 2 ขาย):**  
- `B` = ขายทั้งหมวด (ทั้ง stack)  
- `C` = ขายเฉพาะ common ในหมวด  
- `J` = ขายขยะ common ทั้งกระเป๋า  
→ ยืนยัน y/n · ไม่ขายเรลิก / unique / หีบ / เกียร์ rare+  

**Relic tab (hub `R`):** แยกจากอุปกรณ์ · `divine_burden` / `relic_*`  

sanitize_inventory ยังบังคับ parallel + collapse stack ก่อนเข้า hub

## 9. Anti-patterns

- อย่าแยก list เก็บของคนละ array ถาวร (sync นรก) — **กรองตอนแสดง**  
- อย่าบังคับผู้เล่นจำ id ภาษาอังกฤษ  
- อย่าซ่อน “ใช้ยา” ใต้ชื่อ “อุปกรณ์”

## 10. ตัวอย่างประสบการณ์หลังเปลี่ยน

| ก่อน | หลัง |
|------|------|
| 5 → 1 ใช้ยา (จำลำดับ) | 5 → **2 รักษา** → เลือกยา |
