# ระบบระดับ (Rarity / Grade)

ใช้กับ **อุปกรณ์** · **ปาร์ตี้** (ภูต อสูร เทพ มาร ฯลฯ) · ของดรอป

## ระดับ (ต่ำ → สูง)

| id | ชื่อ | สัญลักษณ์ | คูณสเตตัส (ประมาณ) |
|----|------|-----------|-------------------:|
| common | ธรรมดา | ○ | ×1.00 |
| uncommon | สูง | ◇ | ×1.15 |
| rare | หายาก | ◆ | ×1.35 |
| sacred | ศักดิ์สิทธิ์ | ★ | ×1.55 |
| legendary | ตำนาน | ✦ | ×1.85 |
| divine | เทพ | ✧ | ×2.20 |
| archdivine | มหาเทพ | ✪ | ×2.75 |
| mythic | ปฐมกาล | ◈ | ×3.40 |

ตัวเลขจริงอยู่ที่ `data/rarity/tiers.yaml`

## ผลต่อเกม
- **อุปกรณ์**: ATK/HP/MP จากการสวม × ระดับ (+ อัปเกรด)
- **ปาร์ตี้**: bonus_atk/hp/mana × ระดับตอนรับเข้าทีม
- **ดรอป**: สุ่มระดับ (บอสมีโอกาสสูงกว่า)
- **แสดงผล**: แท็ก `[◆หายาก]` ในกระเป๋า / เกียร์ / ปาร์ตี้ / ดูไอเทม

## ไฟล์
- `data/rarity/tiers.yaml`
- `game/domain/rarity.py`
- เซฟ: `inventory_rarities[]` คู่ `inventory_ids[]` · `equip_rarities`

## มอนสเตอร์ / บอส
- สุ่มระดับตอนเกิด · บอสโน้มสูงกว่า
- HP/ATK/XP × ระดับ (`enemy_threat_mult` / `enemy_xp_mult`)
- ชื่อแสดง เช่น `○หมาป่า` / `✧ราชาป่า (เทพ)`

## ชื่อไอเทมตามระดับ
- ธรรมดา: `ดาบเหล็ก [○ธรรมดา]`
- สูงขึ้น: `ดาบเหล็ก · ตำนาน [✦ตำนาน]`
- ฟังก์ชัน `display_item_name()`

## คราฟกับระดับวัตถุดิบ
```yaml
inputs_rarity:
  iron_sword: uncommon   # ต้อง ≥ สูง
output_rarity: rare
```
- วัตถุดิบระดับต่ำกว่าที่กำหนด **ใช้คราฟไม่ได้**
- ของคุณภาพดีกว่าใช้แทนได้ (หักจากชิ้นระดับต่ำสุดที่ยังพอ)

## ร้าน: ซื้อ-ขายตามระดับ (1.7.2–1.7.3)

### ซื้อ
- ราคา × `price_mult` ของระดับ · สต็อก `{id, rarity}` ได้

### ขาย (S)
1. อ้างอิงราคาซื้อ (หลังคูณระดับ)
2. × `sell_ratio` (ดีฟอลต์ 0.40)
3. **ประกันราคา** — ไม่ต่ำกว่า `price_insurance_floor` × ราคาซื้อ
4. **ภาษีตามระดับ** — ของสูงยิ่งหัก % มาก (`sell_tax_rate`)

### ตลาดเฉพาะระดับ
| ร้าน | ช่วงระดับ |
|------|-----------|
| พ่อค้าเร่ | 1–4 (ถึงศักดิ์สิทธิ์) |
| อาวุธเมือง | 1–5 |
| ตลาดสวรรค์/นรก | 3–7 |
| **ตลาดของหายาก** | 3–5 เท่านั้น |
| **ศาลาตำนาน** | 5–8 เท่านั้น |

ร้านเฉพาะ **ไม่รับซื้อ/ขาย** ของนอกช่วง

```yaml
min_rarity_rank: 5
max_rarity_rank: 8
sell_ratio: 0.48
price_insurance_floor: 0.38
sell_tax_bonus: 0.08
```

### กรอง (ร้านทั่วไป)
A=ทั้งหมด · C=ธรรมดา–สูง · R=หายาก+

## มอนคงที่
ตั้ง `rarity: sacred` ใน `monsters.yaml` — ไม่สุ่ม (ยังคูณสเตตัส)

## ขยาย
เพิ่ม tier / ร้าน specialty ใน YAML ได้
