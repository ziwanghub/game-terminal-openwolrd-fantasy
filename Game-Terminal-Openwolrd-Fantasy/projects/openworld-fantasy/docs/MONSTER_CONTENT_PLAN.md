## 0. หลักการล็อก (สำคัญ)

| # | หลัก | ความหมาย |
|---|------|----------|
| **MC-1** | **จะเติมมอนให้ครบสายเนื้อหาในที่สุด** | ทุกพื้นที่หนาขึ้น · บทบาทครบ · บอส/elite · ดรอป+การ์ดสอดคล้อง |
| **MC-2** | **แต่แบ่งเฟส ค่อยเพิ่ม** | ห้าม dump มอน RO / mob_db ทั้งก้อนทีเดียว · เฟสละ ship + เทส + playtest ได้ |
| **MC-3** | **ยืมบทบาท/ชนิด (แนว RO) ไม่ copy IP** | slime · beast · undead · golem · insect ฯลฯ — **ชื่อโลกเรา** · ห้าม Poring/Baphomet ตรง |
| **MC-4** | **มอนใหม่ = แพ็กครบ** | stats + pool พื้นที่ + `drops` + `card_id` (ถ้ามี) · ไม่ปล่อยมอนเปล่า |
| **MC-5** | **Soft / anti-spoiler** | ไม่โชว์ % · anti_farm · แรงก์มอน soft |
| **MC-6** | **Solo ก่อน online** | ไม่ออกแบบมอนรอบ realtime MP |

### กฎแพ็กมอน 1 ตัว (บังคับทุกเฟส MC1+)

```text
มอนใหม่ต้องมีอย่างน้อย:
  ✓ id · name · level_min/max · hp_base · atk_base · elements
  ✓ อยู่ใน monster_pools ของพื้นที่ ≥1
  ✓ drops: อย่างน้อย 2–4 แถว (mat / ใช้ได้ / เกียร์ soft)
  ✓ card_id + card_rate (หรือใช้การ์ดธาตุชั่วคราว + โน้ตใน IC2)
  ✓ (optional) apply_status · attack_profiles · rarity
  ✓ pick_monster/spawn ต้องเห็น drops (runtime 1.27+)
```

**ห้าม:** เพิ่มมอน 50 ตัวโดยไม่มี drops / ไม่อยู่ pool

## 2. ภาพรวมเฟส (MC0 → MC6)

```text
MC0  กฎ + แผนที่บทบาท/พื้นที่     ← เอกสาร
  │
  ▼
MC1  เติมพื้นที่บาง (เขา·ทะเลทราย)  ← pool 3→5+
  │
  ▼
MC2  เติมบทบาทที่ขาด (โครง RO-like) ← diversity
  │
  ▼
MC3  Elite / ฝูง soft + profiles     ← ไฟต์หลาก
  │
  ▼
MC4  Mid–late + mini-boss            ← ความลึก
  │
  ▼
MC5  ผูก IC (การ์ด·mat·คราฟครบ)      ← เศรษฐกิจปิด
  │
  ▼
MC6  บาลานซ์ทั้งก้อน + ปิดงานมอน     ← “เติมครบ” นิ่ง
```

| เฟส | ชื่อสั้น | เป้าจำนวน (สะสมโดยประมาณ) | เกณฑ์จบเฟส |
|-----|----------|---------------------------|------------|
| **MC0** | กฎ + แผนที่ | doc | ล็อกบทบาท/พื้นที่/กฎแพ็ก |
| **MC1** | เติมพื้นที่บาง | มอน ~48–52 | เขา+ทะเลทราย pool ≥5 · **done 1.28** |
| **MC2** | บทบาทที่ขาด | มอน ~55–62 | อย่างน้อย 6 บทบาทใหม่กระจายพื้นที่ |
| **MC3** | Elite / ไฟต์ | +flag elite · profiles | ไฟต์รู้สึกต่าง · ไม่แค่ HP ฟู |
| **MC4** | Mid–late | มอน ~65–75 · mini 2–4 | late พื้นที่มีตัวเฉพาะ |
| **MC5** | ผูกไอเทม | card_id เฉพาะ · mat ครบ | สอดคล้อง IC2–IC3 |
| **MC6** | ปิดงาน | เป้าปลายทาง | playtest ยาว · spawn/rate นิ่ง |

## 3. รายละเอียดรายเฟส

### MC0 — กฎและแผนที่บทบาท (ไม่ bulk)

| งาน | รายละเอียด |
|-----|------------|
| แผนที่บทบาท (RO-like → เรา) | ดู §4 |
| แผนที่พื้นที่ ← บทบาทที่ขาด | เขาต้องการอะไร · ทะเลทราย · หนอง ฯลฯ |
| กฎแพ็กมอน 1 ตัว | §0 ด้านบน |
| กฎเลเวล/HP/ATK soft band | early / mid / late ช่วงตัวเลขแนวทาง |
| กฎชื่อ | ไทย/โลกเรา · ห้ามชื่อ RO ต้นฉบับ |

**ส่งมอบ:** เอกสารนี้  
**สถานะ:** วางแผน

### MC1 — เติมพื้นที่บางก่อน (ROI สูง)

| พื้นที่ | ตอนนี้ | เป้าเฟส | ตัวอย่างบทบาท (rename โลกเรา) |
|---------|------:|--------:|------------------------------|
| mountain_rock | 3 | **5–6** | นกเขา · แมลงหิน · นักขุด/คนเถื่อนเขา |
| desert_heat | 3 | **5–6** | งูทราย · ฝุ่นวิด · อูฐ/beast เบา |
| (optional) dark_forest | 4 | 5 | พืชกินคน / นกป่า 1 ตัว |

**ต่อมอน:** drops + card_id + ใส่ `monster_pools` weight  
**อย่าทำ:** เพิ่ม void 10 ตัวก่อนเขาแน่น

**เกณฑ์จบ:** เดินเขา/ทะเลทราย 10 ไฟต์ แล้วเจอชนิดหลากกว่าเดิมชัด

---

### MC2 — เติมบทบาทที่ขาด (โครงแบบ RO ไม่ dump)

เลือกบทบาทที่เกมเรายังบาง — **ทีละชุด 6–10 ตัว** ไม่ทั้ง bestiary RO:

| บทบาท (แนว) | ตัวอย่างใช้ในโลกเรา | พื้นที่แนะนำ |
|-------------|---------------------|--------------|
| Plant / ent เบา | พุ่มมีพิษ · เถาวัลย์ | ป่า · หนอง |
| Skeleton / bone | โครงกระดูกซาก | เมือง · ถ้ำ |
| Insect ฝูง | ตัวต่อทราย · มดผลึก | ทะเลทราย · ผลึก |
| Flying | ค้างคาวมีแล้ว · นก/ค้างคาวสายพันธุ์ | เขา · ป่า |
| Aquatic / leech | ปลิงมีแล้ว · ปลาหนอง | หนอง |
| Elemental เล็ก | วิสป์มีแล้ว · เศษไฟ/น้ำแข็ง | ผลึก · รอยแยก |
| Humanoid โจร | นักเลงมีแล้ว · พ่อค้าเถื่อน | เมือง · ทะเลทราย |

**เกณฑ์จบ:** อย่างน้อย 6 บทบาทใหม่ถูกใช้ใน pool จริง · ไม่ซ้ำ id

---

### MC3 — Elite · profiles · ความต่างไฟต์

| งาน | รายละเอียด |
|-----|------------|
| `elite: true` + rarity | บางตัวใน pool weight ต่ำ |
| `attack_profiles` | telegraph soft 2–3 แบบ (มีแบบอย่าง shadow_wraith) |
| `apply_status` | poison/burn/shock เบา — ผูก DD/status ที่มี |
| ฝูง soft | ใช้ pack spawn ที่มี (aoe_balance) · มอน “swarm” id แยกถ้าจำเป็น |

**เกณฑ์จบ:** ไฟต์ mid รู้สึกไม่ใช่แค่ตีกันตรง ๆ ทุกตัว

### MI — ความฉลาดมอน (ไฟต์) · 1.39+

| เฟส | รายละเอียด | สถานะ |
|-----|------------|--------|
| **MI1** | เลือก `attack_profiles` ตามบริบท (เลือดผู้เล่น/สถานะ/ท่ารอบก่อน) เมื่อ `intel_tier`≥1 หรือ elite/boss | **done** `monster_ai.py` |
| **MI2** | soft flee เมื่อฉลาด + HP ต่ำ · บอส/`never_flee` ไม่หนี · ไม่ดรอป | **done** |
| **MI3** | คุย/เจรจา soft ตอนเข้าหา (`rare_talk` + bias) · เมนูสงบ/ของขวัญ/ข่ม/จาก | **done** (1.40) |

YAML เสริม: `intel_tier` · `can_flee` · `never_flee` · `can_talk` (default จาก elite/boss ก็ได้)

### MC4 — Mid–late + mini-boss

| งาน | รายละเอียด |
|-----|------------|
| มอนเฉพาะ crystal / void | 2–4 ตัวต่อพื้นที่ late |
| Mini-boss (optional) | 1 ต่อ 2–3 พื้นที่ · HP/ATK ระหว่างมอน–บอส · drops หนา |
| บอสหลัก 8 ตัว | คง · ปรับ drops/phases ถ้า playtest ชี้ |
| เชื่อม dungeon roster | ถ้าดันใช้ mon id ร่วม |

**เกณฑ์จบ:** late game ไม่วนมอน early ที่ scale อย่างเดียว

### MC5 — ผูกกับแผนไอเทม (IC)

| งาน | คู่ IC |
|-----|--------|
| การ์ดเฉพาะมอนครบสายหลัก | **IC2** |
| Mat ใหม่ต่อมอนชุด MC1–MC2 | **IC3** |
| คราฟใช้ mat นั้น | **IC3** |
| หีบ/บอส pool อ้างของใหม่ | **IC5** |

**กฎ:** เปิด MC5 ได้เมื่อ IC2 อย่างน้อยเริ่มแล้ว (ไม่งั้นการ์ดซ้ำธาตุอย่างเดียว)

**เกณฑ์จบ:** มอนชุด MC1–MC4 มี card_id ไม่ใช่แค่การ์ดไฟ/เงาร่วมกันทั้งแมป

### MC6 — บาลานซ์ทั้งก้อน + ปิดงานมอน

| งาน | รายละเอียด |
|-----|------------|
| จูน weight ใน monster_pools | หายาก/ธรรมดา |
| จูน HP/ATK/xp_mult | เทียบเลเวลผู้เล่น |
| จูน drop rate | กับ IC6 |
| เอกสารแคตตาล็อกมอน | จำนวนต่อพื้นที่ · บทบาท |
| เทส | pick_monster มี drops · data integrity · spawn ทุก area |

**เกณฑ์จบแผน MC:** เป้า ~70–90 มอน · ทุกพื้นที่ pool หนาพอ · แพ็กครบ · playtest 2–4 ชม. ไม่เบื่อซ้ำเร็วเกิน

## 5. ลำดับกับแผนไอเทม (IC) และ Wave

```text
        MC0 ─────────────────────────────┐
        MC1 เติมพื้นที่บาง                │  ขนาน IC1 เกียร์ได้
              │                          │
              ▼                          ▼
        MC2 บทบาทใหม่  ←── ควรใกล้ IC2 การ์ด
              │
              ▼
        MC3 elite/profiles
              │
              ▼
        MC4 late / mini  ←── IC4 mid–late
              │
              ▼
        MC5 ผูกการ์ด/mat  ←── IC2–IC3–IC5
              │
              ▼
        MC6 ปิดงานมอน    ←── IC6 บาลานซ์รวม
```

| ขนานที่แนะนำ | อย่าทำ |
|--------------|--------|
| MC1 ∥ IC1 | dump มอน 40 ตัว + ไอเทม 100 ชิ้นวันเดียว |
| MC2 ∥ IC2 ต้น | มอนใหม่โดยไม่มี drops |
| MC5 หลัง IC2 เริ่ม | การ์ด RO ทั้งก้อน + มอน RO ทั้งก้อน |

**Wave A:** playtest → แล้วเปิด **MC1** คู่ **IC1**  
**อย่า** เริ่ม MC4 late ก่อน early พื้นที่หนา

## 6. คุณภาพทุกเฟส MC1+

1. YAML ใน `data/monsters` + `data/areas` pools  
2. `drops` ครบ · item id มีใน registry  
3. `pick_monster` / boss ยังพก drops (อย่า regress 1.27)  
4. pytest: data integrity · drop table · (optional) pool weight > 0  
5. อัป `PHASES.md` + ตารางสถานะใน `IMPROVEMENT_PLAN.md`  
6. Playtest สั้นหลัง MC1, MC2, MC4, MC6

## 8. Checklist ปลายทาง (“เติมมอนครบ” ตามแผน)

- [ ] ทุกพื้นที่ pool ≥5 (early–mid) / late ≥6

- [ ] บทบาทหลักใน §4 ถูกใช้ครบสาย  
- [ ] ทุกตัวมี drops · ส่วนใหญ่มี card_id เฉพาะ  
- [ ] Elite/profiles มีใน mid+  
- [ ] ไม่มีชื่อ RO ต้นฉบับใน data
