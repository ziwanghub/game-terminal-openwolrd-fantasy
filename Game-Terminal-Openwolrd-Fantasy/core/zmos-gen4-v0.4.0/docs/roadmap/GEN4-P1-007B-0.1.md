## 4. Review Findings

### 4.1 Federation Types

- Node identity, authority owner reference, scope reference, and lineage reference are explicit.
- Simulation result model includes consensus and lineage views for local governance validation only.

### 4.2 Federation Validator

- Node reference validation is strict and fail-closed (`MALFORMED_NODE_REF`, `INVALID_NODE_PARTICIPATION`).
- Trust envelope compatibility is bounded to federation ID + required actions/resources.
- Membership compatibility rejects stale/revoked/failed participation states.
- Lineage mismatch is explicit and fail-closed.
- Scope compatibility rejects missing required actions/resources.

### 4.3 Federation Simulator

- Event stream is deterministically ordered by `event_id` before evaluation.
- Validation failures are captured as structured errors and quarantine is applied.
- Revoked/left membership and inactive audit states force quarantine behavior.
- Simulation remains local and side-effect-free.

### 4.4 Federation Fixtures & Tests

- Fixtures cover valid and invalid trust validation, lineage mismatch, revoked membership, and stale audit state.
- Tests assert deterministic ordering and quarantine outcomes.
- Local validator tests verify fail-closed rejection paths.
